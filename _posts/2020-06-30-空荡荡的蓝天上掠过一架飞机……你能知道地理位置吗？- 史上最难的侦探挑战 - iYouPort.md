  <p class="graf graf--p">
   你看到，什么标志都没有。空荡荡的蓝天，一个没有任何特色的建筑物，一架飞机。
  </p>
  <p class="graf graf--p">
   出题的人提供的唯一信息如下：
  </p>
  <ol class="postList">
   <li class="graf graf--li">
    天空中的飞机是从伦敦飞往香港的波音747
   </li>
   <li class="graf graf--li">
    该照片摄于2019年10月30日。
   </li>
   <li class="graf graf--li">
    拍摄者在旅馆里。
   </li>
  </ol>
  <p class="graf graf--p">
   面临的挑战是要确定拍摄者具体住在哪家旅馆。但是在此之前，还有很多其他东西要首先找到。
  </p>
  <p class="graf graf--p">
   👉照片中还有一些信息也可以帮助您：
  </p>
  <ol class="postList">
   <li class="graf graf--li">
    当时的天气晴朗。
   </li>
   <li class="graf graf--li">
    阴影表明太阳可能在拍摄者的后面。
   </li>
   <li class="graf graf--li">
    飞机在天空中的位置。这点将至关重要。
   </li>
  </ol>
  <p class="graf graf--p">
   如果您尝试通过反向图像搜索解决此问题（开源情报调查的常规思考方式），在这里
   <strong class="markup--strong markup--p-strong">
    完全不管用
   </strong>
   ，您将什么都找不到。
  </p>
  <p class="graf graf--p">
   图片中的建筑物完全没有特点，无法与任何反向图像搜索相匹配。尝试裁剪以寻找匹配也没有任何意义。
  </p>
  <p class="graf graf--p">
   检查发布者的 Twitter 帐户，以查看他在10月30日可能去过的地方，也完全一无所获。如果您曾经尝试过他的任何测验，那么您就会知道他经常旅行。有时在德国、有时在阿尔巴尼亚、甚至在蒙古。也就是说试图猜测他可能在哪个城市也是毫无意义的。
  </p>
  <p class="graf graf--p">
   总之，
   <strong class="markup--strong markup--p-strong">
    几乎理论上的开源情报调查入手捷径都不存在
   </strong>
   。这就是挑战的魅力所在。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     规划一个方法：从大到小
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   那么从哪里开始呢？从建筑物开始可能会花费很长的时间，不如从飞机开始，至少对此有一点了解。总之图片上就这俩东西。
  </p>
  <p class="graf graf--p">
   👉 与往常一样，解决此类地理位置难题的最佳方法是从最大的地方开始，然后逐渐缩小，直到成为一个具体的点。
  </p>
  <p class="graf graf--p">
   从理论上讲，这张照片几乎可以在世界任何地方拍摄，但是通过识别越来越多的信息，这意味着有可能使参数的设定越来越严格，直到识别出特定建筑物为止。
  </p>
  <p class="graf graf--p">
   刚开始时看起来可能不是这样，但是只要能知道飞机的飞行日期和路线，就可以弄清拍摄者在看哪栋建筑物里。
  </p>

  <p class="graf graf--p">
   该建筑可能在世界上的任何地方（在开始阶段这是一个安全的假设……），但我们也知道它在从伦敦到香港之间的飞行路线上。
  </p>
  <p class="graf graf--p">
   👉于是，立刻可以确定，这意味着它不在北美或南美、非洲或大洋洲。它必须位于欧洲、中亚或亚洲本身。
  </p>
  <p class="graf graf--p">
   这看起来似乎还是太大，但是已经缩小了可能的位置范围，超出了您的想象。
  </p>
  <p class="graf graf--p">
   👉我们也知道拍照时是白天。这意味着一旦确定了飞机，就可以确定路线，并且在知道路线后，就可以说出飞机在任何给定时间的位置，以及飞机越过头顶时，该位置是白天还是黑夜。如果此处天黑了，那就可以舍弃其可能性。
  </p>
  <p class="graf graf--p">
   为了做到这一点，必须首先确定飞机。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     识别飞机
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   拍摄者说，这架飞机是10月30日从伦敦飞往香港的，那是一架波音747–8F。
  </p>
  <p class="graf graf--p">
   为了识别飞机，我们需要知道当天伦敦和香港之间往返的所有航班和飞机类型。
  </p>
  <p class="graf graf--p">
   唯一的问题是伦敦有五个国际机场（希思罗机场、盖特威克机场、城市机场、斯坦斯特德机场、和卢顿机场），而香港只有一个。
  </p>
  <p class="graf graf--p">
   👉这意味着查看从任何伦敦机场到达香港的航班将更容易，而不是尝试以五个可能的起点进行相反的推测。（这是小窍门）
  </p>
  <p class="graf graf--p">
   历史航班数据可从几个不同的地方获得。
   <a class="markup--anchor markup--p-anchor" data-href="https://www.flightradar24.com/" href="https://www.flightradar24.com/" rel="noopener noreferrer" target="_blank">
    FlightRadar
   </a>
   是其中最知名的，但也有
   <a class="markup--anchor markup--p-anchor" data-href="https://www.airportia.com/" href="https://www.airportia.com/" rel="noopener noreferrer" target="_blank">
    Airportia
   </a>
   和
   <a class="markup--anchor markup--p-anchor" data-href="https://flightaware.com/" href="https://flightaware.com/" rel="noopener noreferrer" target="_blank">
    Flight Aware
   </a>
   等可用。
  </p>
  <p class="graf graf--p">
   FlightAware 会针对此类历史数据收费，而 Airportia 仅列出定期的客运航班。幸运的是，
   <a class="markup--anchor markup--p-anchor" data-href="https://twitter.com/TobiTweetings/status/1189610407079694336" href="https://twitter.com/TobiTweetings/status/1189610407079694336" rel="noopener noreferrer" target="_blank">
    Tobi Bluhm
   </a>
   已经确定该航班一定是
   <a class="markup--anchor markup--p-anchor" data-href="https://www.flightradar24.com/data/aircraft/n617up" href="https://www.flightradar24.com/data/aircraft/n617up" rel="noopener noreferrer" target="_blank">
    N617UP
   </a>
   ，这是UPS的货运航班，于10月30日世界标准时间6:12从伦敦斯坦斯特德机场（Stansted）出发，飞往香港。也就是说很幸运，这里可以省钱了（花钱在 FlightAware 上查到这点并不难）。
  </p>

  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     使用KML文件绘制路径
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   从理论上讲，所有这一切都足够容易，但是如何才能从 6000 英里的飞行路线中找到确切的酒店呢？
  </p>
  <p class="graf graf--p">
   幸运的是，FlightRadar 和 FlightAware 都可以将飞行轨迹下载为KML文件。KML文件只是包含有关一系列地理位置（例如GPS坐标和海拔高度等）数据的文件。
  </p>
  <p class="graf graf--p">
   你可以在 Google Earth 中打开该文件，然后可以查看和分析 KML 映射数据。这意味着将能够在飞机旅程的每个阶段看到飞机的确切位置、高度和方向。
  </p>
  <p class="graf graf--p">
   👉在 Google Earth 中查看这些数据将有助于直观地看到飞机所在的位置，然后通过将 Google Earth 中的视图与照片中的信息进行交叉引用，就可以确定拍摄者的位置。
  </p>
  <p class="graf graf--p">
   要获得任何航班的 KML 文件，您需要一个 Flightradar 付费帐户，或者您可以注册 FlightAware 以获得为期
   <strong class="markup--strong markup--p-strong">
    三个月的免费试用
   </strong>
   ，该服务将允许访问KML文件。
  </p>
