  <p class="graf graf--p">
   安全研究人员
   <a href="https://www.martinvigo.com/email2phonenumber/" rel="noopener noreferrer" target="_blank">
    Martin Vigo
   </a>
   对密码重置选项中的弱点和攻击媒介很在行，他谈到了一个名为 “Ransombile” 的工具。
  </p>
  <p class="graf graf--p">
   该工具可以通过短信为许多Alexa排名前100的网站
   <strong class="markup--strong markup--p-strong">
    自动执行
   </strong>
   密码重置过程，⚠️有助于进行有针对性的攻击。
  </p>
  <p class="graf graf--p">
   <iframe allowfullscreen="allowfullscreen" height="421" src="//www.youtube.com/embed/7MbgFTJrhss" width="750">
   </iframe>
  </p>
  <p class="graf graf--p">
   他还谈到了滥用 DEF CON 和CCC上的语音信箱系统所带来的广泛影响，该滥用是通过电话重置密码。
  </p>
  <p class="graf graf--p">
   <iframe allowfullscreen="allowfullscreen" height="421" src="//www.youtube.com/embed/ZpZbTPYB3cU" width="750">
   </iframe>
  </p>
  <p class="graf graf--p">
   在研究这些主题时，Martin Vigo 花了许多时间在各种不同的网站上测试和重置密码。他开始注意到一种以前从未注意到的模式。
  </p>
  <p class="graf graf--p">
   当您想重设密码时，你会看到不同的选项；通常包括：接收带有唯一链接的电子邮件，以进行单击验证；获取带有六位数代码的SMS，甚至可以选择直接接听电话并听到密码。
  </p>
  <p class="graf graf--p">
   在查看通过短信或电话重置密码的选项时，你会注意到，用户界面上通常显示部分电话号码。
  </p>
  <p class="graf graf--p">
   但是，它的屏蔽方式是仅显示其中几位数字，但是足以失败用户。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    换句话说，如果我知道您的电子邮件，我可以为您的帐户启动密码重置过程，并获取您的电话号码的至少几位数
   </strong>
   。这就是攻击的开端。
  </p>
  <p class="graf graf--p">
   如上所述，他花了很多时间重置密码，但意识到并非所有网站都显示相同的数字。有些会显示最后四位，有些会显示第一位和最后两位，依此类推。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    ⚠️没有掩盖电话号码等个人身份信息（PII）的标准方法
   </strong>
   。如何遮蔽完全由开发人员决定，这就是问题所在。
  </p>
  <p class="graf graf--p">
   为了说明实际情况，以贝宝（Paypal）为例。
  </p>
  <p class="graf graf--p">
   <img alt="" class="aligncenter size-full wp-image-9332 jetpack-lazy-image" data-lazy-sizes="(max-width: 600px) 100vw, 600px" data-lazy-src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?resize=600%2C753&amp;is-pending-load=1#038;ssl=1" data-lazy-srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?w=600&amp;ssl=1 600w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?resize=239%2C300&amp;ssl=1 239w" data-recalc-dims="1" height="753" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?resize=600%2C753&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="600"/>
   <noscript>
    <img alt="" class="aligncenter size-full wp-image-9332" data-recalc-dims="1" height="753" sizes="(max-width: 600px) 100vw, 600px" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?resize=600%2C753&amp;ssl=1" srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?w=600&amp;ssl=1 600w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/1-7.jpg?resize=239%2C300&amp;ssl=1 239w" width="600"/>
   </noscript>
  </p>
  <p class="graf graf--p">
   如果我是攻击者，启动密码重置过程，它将显示第一位和最后四位。
  </p>
  <p class="graf graf--p">
   <img alt="" class="aligncenter size-full wp-image-9333 jetpack-lazy-image" data-lazy-sizes="(max-width: 400px) 100vw, 400px" data-lazy-src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?resize=400%2C513&amp;is-pending-load=1#038;ssl=1" data-lazy-srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?w=400&amp;ssl=1 400w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?resize=234%2C300&amp;ssl=1 234w" data-recalc-dims="1" height="513" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?resize=400%2C513&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="400"/>
   <noscript>
    <img alt="" class="aligncenter size-full wp-image-9333" data-recalc-dims="1" height="513" sizes="(max-width: 400px) 100vw, 400px" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?resize=400%2C513&amp;ssl=1" srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?w=400&amp;ssl=1 400w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/09/2-7.jpg?resize=234%2C300&amp;ssl=1 234w" width="400"/>
   </noscript>
  </p>
  <p class="graf graf--p">
   但是，如果我登录并遇到2FA的挑战，它将仅显示最后三位。
  </p>
  <p class="graf graf--p">
   这意味着，仅用您的电子邮件地址，我就能获得您电话号码中的五位数。如果我知道您的电子邮件和密码，那么我只会得到三位数 。
  </p>
  <p class="graf graf--p">
   与仅知道您的电子邮件地址的攻击者相比，贝宝对已经知道您的密码的攻击者隐藏了更多的数字。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    深层发掘
   </strong>
  </p>
  <p class="graf graf--p">
   列出人们倾向于注册的热门网站，并检查这些网站的密码重置过程。
  </p>
  <p class="graf graf--p">
   目标是确定哪些站点只会要求电子邮件来启动该过程（无需进一步的信息），支持基于移动设备的密码重置、以及“泄漏”的电话号码位数。
  </p>
  <p class="graf graf--p">
   以下是一小部分。
  </p>
  <p class="graf graf--p">
   泄漏前三位和后两位：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    eBay
   </li>
  </ul>
  <p class="graf graf--p">
   泄漏第一位和后四位：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    Paypal
   </li>
  </ul>
  <p class="graf graf--p">
   泄漏第一位和后两位：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    Yahoo
   </li>
  </ul>
  <p class="graf graf--p">
   泄漏最后四位：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    Lastpass
   </li>
  </ul>
  <p class="graf graf--p">
   泄漏最后两位：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    Google
   </li>
   <li class="graf graf--li">
    Facebook
   </li>
   <li class="graf graf--li">
    Twitter
   </li>
   <li class="graf graf--li">
    Hotmail
   </li>
   <li class="graf graf--li">
    Steam
   </li>
  </ul>
  <p class="graf graf--p">
   查看上面的列表可以得出结论，例如，⚠️
   <strong class="markup--strong markup--p-strong">
    如果您同时拥有 eBay 和 LastPass 帐户，则攻击者就可以知道您的电话号码中十分之七的数字（美国电话号码10位数）。
   </strong>
  </p>
  <p class="graf graf--p">
   换句话说，攻击者可以仅仅使用您的电子邮件地址猜测到您电话号码的可能性，已经从十亿个可能性降低到一千个。
  </p>
  <p class="graf graf--p">
   这不是唯一可能的组合，但在这篇文章中重点介绍这种情况。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    发现剩余的数字
   </strong>
  </p>
  <p class="graf graf--p">
   十分之七的数字，这意味着现在只缺少三个数字。在这一点上，
   <strong class="markup--strong markup--p-strong">
    重要的是集中于我们已经知道哪些数字
   </strong>
   。
  </p>
  <p class="graf graf--p">
   美国电话号码由3个字段组成：区号（或NPA），电话局（或中心局代码）和用户号码。当然还有国家/地区代码，但暂时专注于美国号码（后面扩展到其他国家 —— 攻击模式是一样的）。
  </p>
  <p>
   <img alt="" class="aligncenter size-full wp-image-9334 jetpack-lazy-image" data-lazy-sizes="(max-width: 768px) 100vw, 768px" data-lazy-src="https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?resize=768%2C194&amp;is-pending-load=1#038;ssl=1" data-lazy-srcset="https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?w=768&amp;ssl=1 768w, https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?resize=300%2C76&amp;ssl=1 300w" data-recalc-dims="1" height="194" src="https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?resize=768%2C194&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="768"/>
   <noscript>
    <img alt="" class="aligncenter size-full wp-image-9334" data-recalc-dims="1" height="194" sizes="(max-width: 768px) 100vw, 768px" src="https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?resize=768%2C194&amp;ssl=1" srcset="https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?w=768&amp;ssl=1 768w, https://i1.wp.com/www.iyouport.org/wp-content/uploads/2019/09/3-5.jpg?resize=300%2C76&amp;ssl=1 300w" width="768"/>
   </noscript>
  </p>
  <p class="graf graf--p">
   eBay + LastPass 泄漏了区号和用户号码。重要的是要强调，这丢失的3位数字是中心局代码。
  </p>
  <p class="graf graf--p">
   这很重要，因为它将帮助你进一步缩小可能性。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    NANPA
   </strong>
  </p>
  <p class="graf graf--p">
   <a class="markup--anchor markup--p-anchor" data-href="https://www.nationalnanpa.com/" href="https://www.nationalnanpa.com/" rel="noopener noreferrer" target="_blank">
    NANPA
   </a>
   是北美编号计划管理员，加拿大、美国和某些加勒比国家的公用电话交换网的编号方案。
   <strong class="markup--strong markup--p-strong">
    这个网站是金矿
   </strong>
   ！
  </p>
  <p class="graf graf--p">
   从这个来源中可以学到了很多电话系统的工作原理。最重要的是，能找到你想要的东西。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    NANPA 维护着实时更新的区号列表、以及
   </strong>
   <a class="markup--anchor markup--p-anchor" data-href="https://www.nationalnanpa.com/reports/reports_cocodes_assign.html" href="https://www.nationalnanpa.com/reports/reports_cocodes_assign.html" rel="noopener noreferrer" target="_blank">
    <strong class="markup--strong markup--p-strong">
     可公开访问的
    </strong>
   </a>
   <strong class="markup--strong markup--p-strong">
    相应中心局代码
   </strong>
   。它会经常更新，您可以查询数据或下载包含所有信息的可分析文件。
  </p>

  <p class="graf graf--p">
   那么这个东西怎么用呢？让我们以旧金山的区号415为例。如果我只缺少对应于中心局代码的三个数字，那么我有1000个可能的数字匹配目标人。
  </p>
  <p class="graf graf--p">
   通过使用 NANPA 数据集，我可以将其减少为784个可能的数字，因为有216个交换号未分配给415区号。这就减少了20％以上，还不错！
  </p>
  <p class="graf graf--p">
