  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     配置
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   Spiderfoot 可在 Linux、Mac OS 和 Windows 上运行。它使用 Python 2.7（已在 Linux 和 Mac OS 中预装），但是要在 Windows 中进行设置，您需要首先做一些其他事。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    Windows
   </strong>
  </p>
  <p class="graf graf--p">
   Spiderfoot 2.12 版是预编译的exe文件，您可以
   <a class="markup--anchor markup--p-anchor" data-href="https://www.spiderfoot.net/documentation/#windows" href="https://www.spiderfoot.net/documentation/#windows" rel="noopener noreferrer" target="_blank">
    在此处
   </a>
   下载并单击以运行。
  </p>
  <p class="graf graf--p">
   如果您在 Windows 环境中工作，这是启动 Spiderfoot 的一种简便方法，但是请注意，更高版本的 Spiderfoot 将无法通过这种方式工作，因此您必须安装适用于Windows的Python和Pip并从
   <a class="markup--anchor markup--p-anchor" data-href="https://github.com/smicallef/spiderfoot" href="https://github.com/smicallef/spiderfoot" rel="noopener noreferrer" target="_blank">
    Github 存储库
   </a>
   中保持更新。
  </p>
  <p class="graf graf--p">
   如果您对Git、Pip和命令行不完全满意，这是在 Windows 中开始使用 Spiderfoot 的最简单方法。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    MacOS and Linux
   </strong>
  </p>
  <p class="graf graf--p">
   运行 Spiderfoot 并使它保持最新状态的最佳方法是使用
   <a class="markup--anchor markup--p-anchor" data-href="https://docs.gitlab.com/ee/gitlab-basics/start-using-git.html" href="https://docs.gitlab.com/ee/gitlab-basics/start-using-git.html" rel="noopener noreferrer" target="_blank">
    git
   </a>
   ，并按照以下说明克隆
   <a class="markup--anchor markup--p-anchor" data-href="https://github.com/smicallef/spiderfoot" href="https://github.com/smicallef/spiderfoot" rel="noopener noreferrer" target="_blank">
    Spiderfoot Github 存储库
   </a>
   。
  </p>
  <p class="graf graf--p">
   首先，请检查您的设备上是否已安装git，因为默认情况下未在所有发行版中安装git。要检查是否存在，请在控制台中键入以下内容：
  </p>
  <pre class="graf graf--pre"><code>git --version</code></pre>
  <p class="graf graf--p">
   如果安装了git，您将收到一条消息，告诉您它是哪个版本。
  </p>
  <p class="graf graf--p">
   如果系统提示您不存在git，请按照以下说明进行安装：
  </p>
  <pre class="graf graf--pre"><code>sudo apt install git</code></pre>
  <p class="graf graf--p">
   如果您使用的不是基于 Debian 的 Linux，则需要使用系统自带的软件包管理器，而不是 apt。
  </p>
  <p class="graf graf--p">
   如果您使用的是 Mac OS，
   <a class="markup--anchor markup--p-anchor" data-href="http://osxdaily.com/2018/03/07/how-install-homebrew-mac-os/" href="http://osxdaily.com/2018/03/07/how-install-homebrew-mac-os/" rel="noopener noreferrer" target="_blank">
    请先安装 brew
   </a>
   （默认情况下未在Mac OS上安装 brew），然后在终端中输入以下内容：
  </p>
  <pre class="graf graf--pre"><code>brew install git</code></pre>
  <p class="graf graf--p">
   然后您就完成了。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    无论您使用的是 Linux 还是 Mac，从此刻开始，其余安装说明都是相同的
   </strong>
   。
  </p>
  <p class="graf graf--p">
   在终端中输入：
  </p>
  <pre class="graf graf--pre"><code>git clone https://github.com/smicallef/spiderfoot.git</code></pre>
  <p class="graf graf--p">
   这将在您的主目录中创建一个名为“spiderfoot”的目录。在继续下一步之前，通过在控制台中输入以下内容来进行移动：
  </p>
  <pre class="graf graf--pre"><code>cd spiderfoot</code></pre>
  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     安装 Pip Requirements
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   如果您以前曾尝试过从GitHub安装 Python 脚本，你会知道很容易陷入困境。通常，安装指南会告诉您输入以下内容：
  </p>
  <pre class="graf graf--pre"><code>pip install -r requirements.txt</code></pre>
  <p class="graf graf--p">
   该命令的作用是通读 Python 文件列表，列表在名为 requirements.txt 的文件中列出，然后尝试安装它们。
  </p>
  <p class="graf graf--p">
   这听起来很容易，但此时人们经常会遇到麻烦，被报错，因此无法安装软件包。
  </p>
  <p class="graf graf--p">
   最常见的原因是安装者的操作系统使用的 Python 和 Pip 版本与程序所需的版本不同。
  </p>
  <p class="graf graf--p">
   系统的默认 Python 版本可能是 Python 3.6，因此Pip假定您要下载与该版本 的 Python 兼容的软件包。
  </p>
  <p class="graf graf--p">
   这可能会导致很多冲突，并使您的软件无法正常工作。某些最受欢迎的 Python OSINT 工具并非都可以在同一版本的Python上运行。
  </p>
  <p class="graf graf--p">
   避免这种情况的方法是确保在调用Pip时使用正确版本的 Python。
  </p>
  <p class="graf graf--p">
   对于 Spiderfoot 来说它就是 Python 2.7。如果您的系统上安装了更新版本的 Python，就可能会报错。
  </p>
  <p class="graf graf--p">
   要检查系统使用的Python版本，请在终端中输入以下内容：
  </p>
  <pre class="graf graf--pre"><code>python --version</code></pre>
  <p class="graf graf--p">
   然后，控制台将告诉您系统默认使用哪个Python版本。
  </p>
  <p class="graf graf--p">
   如果您要安装 Spiderfoot 并且 Python 版本高于2.7，则需要执行以下操作：
  </p>
  <p class="graf graf--p">
   移至您之前创建的 spiderfoot 目录，然后输入以下内容：
  </p>
  <pre class="graf graf--pre"><code>python2.7 -m pip install -r requirements.txt</code></pre>
  <p class="graf graf--p">
   以这种方式输入命令可确保您启动 Python 2.7，使用 -m 为该版本的 Spiderfoot 启动适当的Pip版本。
  </p>
  <p class="graf graf--p">
   安装完成您就可以准备就绪了。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     开始使用 Spiderfoot
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   完成安装后，您将可以首次启动 Spiderfoot。
  </p>
  <p class="graf graf--p">

  <p class="graf graf--p">
   blocklist.de 和 AbuseIPDB 都找到了该IP地址的匹配项。单击 AbuseIPDB 链接可获取更多信息：
  </p>
  <p class="graf graf--p">
   <img alt="" class="aligncenter size-full wp-image-9766 jetpack-lazy-image" data-lazy-sizes="(max-width: 1100px) 100vw, 1100px" data-lazy-src="https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=1100%2C635&amp;is-pending-load=1#038;ssl=1" data-lazy-srcset="https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?w=1175&amp;ssl=1 1175w, https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=300%2C173&amp;ssl=1 300w, https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=768%2C443&amp;ssl=1 768w, https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=1024%2C591&amp;ssl=1 1024w, https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=1100%2C635&amp;ssl=1 1100w" data-recalc-dims="1" height="635" src="https://i2.wp.com/www.iyouport.org/wp-content/uploads/2019/10/8-4.png?resize=1100%2C635&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="1100"/>

  <p class="graf graf--p">
   知道这一点很有用 —— 可以确信防火墙规则已设置并正常工作，并且 Spiderfoot 产生的结果不是误报。

   几分钟后，Spiderfoot 发现了更多信息：

  <p class="graf graf--p">
   与所有搜索结果一样，那里也可能存在一些误报；但在大约十五分钟内，Spiderfoot 就收集到了大量的信息，如果您手动做这件事，可能需要数小时甚至数天。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     示例2 — 研究电子邮件地址
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   仅作为示例，这里将搜索亚马逊大佬 Jeff Bezos 的电子邮件地址，并查看 Spiderfoot 返回的内容。

  <p class="graf graf--p">

  <p class="graf graf--p">
   撰写此类文章的部分困难在于，如果尝试使用现实世界中的示例，很可能会使某些无辜的人感到惊讶。那么还是用 Bezos 吧。

  <p class="graf graf--p">
   Spiderfoot 返回了相关
   <strong class="markup--strong markup--p-strong">
    用户名、数据泄露、PGP密钥，甚至是页面源代码中包含目标电子邮件地址的网站
   </strong>
   。
  </p>
  <p class="graf graf--p">
   一如既往地检查所有内容，并注意误报的可能性。
  </p>
  <p class="graf graf--p">
   Spiderfoot 的真正优势在于，通过开源情报查询的快速自动化，它可以节省大量时间，并带回大量有用的数据，从而使您有更多时间专注于情报分析和调查报告方面。
  </p>
  <p class="graf graf--p">
   用它定期搜索自己，可以帮助您更加有针对性地采取防御措施。
  </p>
  <p class="graf graf--p">
   Spiderfoot 的创建者是 Steve Micallef。这是一个伟大的工程。
  </p>
