   下面开始。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    重建隐藏的朋友列表
   </strong>
  </p>
  <p class="graf graf--p">
   Facebook 允许其用户决定是否在他们的个人资料页面显示他们自己的朋友、以及向谁显示。在设置页面中，它还会显示以下警告：
  </p>
  <blockquote class="graf graf--blockquote">
   <p>
    Remember, your friends control who can see their friendships on their own Timelines. If people can see your friendship on another timeline, they’ll be able to see it in News Feed, search and other places on Facebook. If you set this to Only me, only you will be able to see your full friends list on your timeline. Other people will see only mutual friends.
   </p>
  </blockquote>
  <p class="graf graf--p">
   这是因为 Facebook 还提供“共同朋友”功能，前提是其中至少有一个人的“朋友”列表对当前用户可见。
  </p>
  <p class="graf graf--p">
   这个问题多年来一直为人所知，但该公司不关注这个问题。
  </p>
  <p class="graf graf--p">
   该功能位于URL：
   <em class="markup--em markup--p-em">
    https://www.facebook.com/browse/mutual_friends/?uid=
   </em>
   <strong class="markup--strong markup--p-strong">
    <em class="markup--em markup--p-em">
     ID1
    </em>
   </strong>
   <em class="markup--em markup--p-em">
    &amp;node=
   </em>
   <strong class="markup--strong markup--p-strong">
    <em class="markup--em markup--p-em">
     ID2；
    </em>
   </strong>
   其中 ID1 和 ID2 是两个 Facebook 帐户的数字ID。稍后我们将介绍如何找到任何用户的 Facebook ID。
  </p>
  <p class="graf graf--p">
   请注意，
   <strong class="markup--strong markup--p-strong">
    Facebook 返回的结果既不完整也不一致
   </strong>
   。
  </p>
  <p class="graf graf--p">
   从不同位置或帐户调用相同的功能可能会产生不同的结果。
  </p>
  <p class="graf graf--p">
   由于“共同朋友”的功能，可以重建目标用户的好友列表的很大一部分：给定一个已知当前用户可以看到他们的朋友列表的帐户，并且，至少有一个与目标用户的共同朋友（称为
   <strong class="markup--strong markup--p-strong">
    枢纽用户
   </strong>
   ），从他们的共同朋友开始，
   <strong class="markup--strong markup--p-strong">
    可以迭代地应用共同朋友功能，使用在每个步骤中找到的所有“新朋友”作为另一个迭代的输入，直到达到函数的固定点为止
   </strong>
   。
  </p>
  <p class="graf graf--p">
   为了展示这种方法的潜力，Mattia Vinci 编写了一个简单的 Python 工具，它将
   <strong class="markup--strong markup--p-strong">
    自动执行上述过程
   </strong>
   。
  </p>
  <p class="graf graf--p">
   请注意，Facebook 服务条款禁止未经许可进行抓取：因此，未经适当授权，您不得使用该工具。
  </p>
  <p class="graf graf--p">
   该工具名为 ffff，可以
   <a class="markup--anchor markup--p-anchor" data-href="https://github.com/sowdust/ffff" href="https://github.com/sowdust/ffff" rel="noopener noreferrer" target="_blank">
    从Github下载
   </a>
   。
  </p>
  <p class="graf graf--p">
   它是用 Python 3 编写的，并使用 selenium 库和 Mozilla geckodriver 来打开浏览器并递归访问共同朋友页面。
  </p>
  <p class="graf graf--p">
   它会继续跟踪找到的关系，以便构建关系网络图（稍后会有详细介绍）。
  </p>
  <p class="graf graf--p">
   ⚠️为了不被 Facebook 检测到抓取，该工具在每个请求之间暂停一个随机的秒数；这个间隔可以减少，但如果太少就有被平台阻挡的风险。
  </p>
  <p class="graf graf--p">
   在该过程结束时，它会生成一个csv文件，其中包含找到的朋友列表，以及一个包含相关图表的gexf文件。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    建立人际关系网络图
   </strong>
  </p>
  <p class="graf graf--p">
   在进行上述调查的过程中你还可以
   <strong class="markup--strong markup--p-strong">
    保持跟踪目标的朋友之间存在的关系
   </strong>
   。
  </p>
  <p class="graf graf--p">
   如果你发现目标用户T和枢纽用户P有一个共同的朋友C，那么，现在你不仅知道C是目标的朋友，而且P和C都是朋友。
  </p>
  <p class="graf graf--p">
   迭代这个过程，你就能够
   <strong class="markup--strong markup--p-strong">
    构建一个沿途找到的所有人际关系的网络图
   </strong>
   ，其中节点是用户，连线是现有的关系。
  </p>
  <p class="graf graf--p">
   一旦有了图表，就可以在其上
   <strong class="markup--strong markup--p-strong">
    运行不同的算法
   </strong>
   ，以便尝试识别网络中的现有社区。例如，在实际情况中，这可能有助于识别工作同事、家人、最亲密朋友等不同的社区。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    实际案例：扎克伯格的朋友圈
   </strong>
  </p>
  <p class="graf graf--p">
   举个例子，我们将尝试获取一些 Mark Zuckerberg 的 Facebook 朋友，他们肯定知道这种可能性。
  </p>
  <p class="graf graf--p">
   请注意，您显然可以使用公共朋友列表构建用户的关系图：在这种情况下，您不需要透视用户；只需使用他/她的 Facebook ID 作为目标和支点。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    找到目标的 Facebook ID
   </strong>
  </p>
  <p class="graf graf--p">
   首先需要了解 Mark Zuckerberg 的 Facebook ID。为了找到 Facebook 帐户的数字ID，有多种方法：
   <a class="markup--anchor markup--p-anchor" data-href="https://www.google.com/search?q=find+facebook+user+id" href="https://www.google.com/search?q=find+facebook+user+id" rel="noopener noreferrer" target="_blank">
    简单的 Google 搜索
   </a>
   就可以返回许多相关在线服务。
  </p>
  <p class="graf graf--p">
   不过，本指南最喜欢的方法是
   <strong class="markup--strong markup--p-strong">
    自主执行
   </strong>
   ：在浏览器中打开目标的 Facebook 个人资料页面（在示例中是
   <a class="markup--anchor markup--p-anchor" data-href="https://www.facebook.com/zuck" href="https://www.facebook.com/zuck" rel="noopener noreferrer" target="_blank">
    Mark Zuckerberg
   </a>
   ）并检查HTML源码（Ctrl + U）。
  </p>
  <p class="graf graf--p">
   在代码中，搜索（Ctrl + F）： fb：// profile /; 应该有两个匹配，后跟相同的整数值：

   </strong>

   你还需要找到至少

   </strong>

    一个与你的调查目标相关的共同朋友
   </strong>
   。
  </p>
  <p class="graf graf--p">
   通常，一个便捷的方法是查看与目标进行交互的用户，例如通过评论或回应他帖子的人。
  </p>

   还要注意，为了最大化找到的“朋友”的数量，

   </strong>

  </p>

    ffff项目的存储库中
   </a>
   。
  </p>

    Github上的项目页面
   </a>

  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    绘制社区图表
   </strong>
  </p>
  <p class="graf graf--p">
   接下来将使用 Gephi 软件绘制扎克伯格的朋友关系图表，可能以某种方式反映关系网络中存在的
   <strong class="markup--strong markup--p-strong">
    子社区的结构
   </strong>
   。
  </p>
  <p class="graf graf--p">
   这里绝不是 Gephi 的指南，也不是进行社区识别的最佳方式。这里只为了演示工具的使用方法。
  </p>
  <p class="graf graf--p">
   首先，用 Gephi 打开 .gexf 文件。最初，它看起来像一堆凌乱的黑暗的线条和圆点。
  </p>

   从这里可以应用一些
   <strong class="markup--strong markup--p-strong">
    布局算法
   </strong>
   ，这些算法可能有助于更深入地了解被视为社区的图表。
  </p>
  <p class="graf graf--p">
   可以从“Overview tab”选项卡的左下窗口中选择布局。每个布局都有不同的参数：我建议都玩一下，看看它们的效果。
  </p>
  <p class="graf graf--p">
   本指南先运行 OpenOrd，然后运行 ForceAtlas 2。
  </p>
  <p class="graf graf--p">
   对于后者，我将 “Scaling” 值设置为10。结果尚不具有吸引力，但具有不同的形状。
  </p>

   现在可以尝试使用 “Modularity” 将节点划分到不同的社区，可以通过屏幕右侧的 “statistics” 部分计算。
  </p>
  <p class="graf graf--p">
   在这里，你可以为“resolution”参数提供一个值：值越小，分区越多（即：更多社区）。
  </p>
  <figure aria-describedby="caption-attachment-9227" class="wp-caption aligncenter" id="attachment_9227" style="width: 1278px">

  <p class="graf graf--p">
   现在我们可以为每个”Modularity”
   <strong class="markup--strong markup--p-strong">
    分配不同的颜色
   </strong>
   。
  </p>
  <p class="graf graf--p">
   在左上角的“外观”选项卡中，选择“节点”&gt;“分区”，并选择“Modularity Class”作为属性。
   <strong class="markup--strong markup--p-strong">
    这是为了更容易看懂
   </strong>
   。
  </p>
  <p class="graf graf--p">
   单击“Palette …”可以配置要使用的颜色。
  </p>
  <p class="graf graf--p">
   单击“应用”所有节点将根据你定义的 Modularity Class 进行着色。如果您对找到的社区不满意，可以返回上一步并选择不同的参数。
  </p>
  <p class="graf graf--p">
   如果对图表感到满意，请转到顶部菜单中的“预览”选项卡，然后配置最终图形的显示方式。
  </p>
  <p class="graf graf--p">
   最后，您可以
   <strong class="markup--strong markup--p-strong">
    以您希望的格式导出它
   </strong>
   。
  </p>
