  </p>
  <p class="graf graf--p">
   许多开源情报初级从业者认为，使用VPN服务足以隐藏其在线跟踪 —— 这是完全
   <strong class="markup--strong markup--p-strong">
    错误
   </strong>
   的。
  </p>
  <p class="graf graf--p">
   使用可靠的VPN只会隐藏您的IP地址，而对于跟踪您的任何跟踪器来说，您的其他数字足迹都是显而易见的。
  </p>
  <p class="graf graf--p">
   完全的在线匿名性非常难以实现。要成为完全匿名的在线状态，您需要使用一套工具和策略来隐藏任何可以揭示您的身份甚至是用于访问互联网的硬件和连接类型的踪迹，并且这需要一些认真的技术技能。
  </p>
  <p class="graf graf--p">
   只有与外国间谍活动有关的那种等级的案件才需要这种匿名性，通常由非常了解如何隐瞒其搜集活动的情报机构进行。而不是公民调查者。
  </p>

   </strong>

   </strong>
   。

  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     什么是在线跟踪？
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   在线跟踪可以定义为追踪记录跨不同网站的互联网用户的互联网浏览历史记录、有时是在线行为的过程。
  </p>
  <p class="graf graf--p">
   为了将浏览历史链接到目标用户，将使用标识符将每个在线用户区分为可以跟踪的个人。
  </p>
  <p class="graf graf--p">
   该标识符类似于人的指纹，因为它可以在数百万个已连接的用户中区分出特定的用户设备。
  </p>
  <p class="graf graf--p">
   下一节将演示在线跟踪在技术上如何工作。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #ff6600;">

   在线跟踪通常利用以下一种或多种方法：
  </p>
  <p class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     1、IP地址跟踪
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   没有IP就无法联网，IP地址是唯一的标识符，用于在连接到互联网时标识任何具有联网功能的设备。
  </p>
  <p class="graf graf--p">
   没有两个设备可以在同一IP网络上拥有相同的IP地址，这使得IP地址成为在线跟踪用户的首选。
  </p>
  <p class="graf graf--p">
   连接到 Internet 时，您要么每次使用相同的IP地址（静态IP），要么每次使用不同的数字（称为动态IP）。
  </p>
  <p class="graf graf--p">
   静态IP地址是您的互联网服务提供商（ISP）分配的地址，不会随时间变化；相反，每当您连接到互联网时，ISP都会分配一个动态IP地址。
  </p>
  <p class="graf graf--p">
   每次重新启动计算设备或路由器时，它都使用一种被称为动态主机配置协议（DHCP）的东西为您分配新的IP地址。一些ISP可能会多次分配以前分配给您的相同IP地址，但这不是经验法则。
  </p>
  <p class="graf graf--p">
   请注意，使用不同的技术（例如VPN和TOR网络之类的匿名网络）上线时，可以伪造（隐藏）IP地址。用户还可以使用NAT路由器，它使所有属于同一网络的设备都共享一个公共IP地址。
  </p>

  <p class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     2、Cookie 追踪
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   Cookies 是跟踪在线用户的最常用技术，如今家喻户晓。
  </p>
  <p class="graf graf--p">
   Cookie 是用户访问特定网站时创建的小型文本文件，其中包含的标准信息包括区分客户端设备的唯一ID、有效日期、和Cookie网站名称。
  </p>
  <p class="graf graf--p">
   当再次返回同一网站时，cookie 用于判断客户端设备。网站使用 Cookie 的主要目的有两个：存储登录凭据、和跟踪用户的在线行为。
  </p>
  <p class="graf graf--p">
   大多数人在谈论 Web cookie 时都意味着一个基本的 cookie 文件（也称为 HTTP cookie）。HTTP cookie 是一个简单的文本文件，用于跟踪用户对部署它的网站的访问。
  </p>
  <p class="graf graf--p">
   关闭浏览器时，过期的 HTTP cookie 会自动删除。但是，到期日期可能是未来的
   <strong class="markup--strong markup--p-strong">
    很多年
   </strong>
   。就 cookie 的寿命而言，主要有两种类型：会话cookie和持久cookie。
  </p>
  <p class="graf graf--p">
   会话Cookie存储在临时内存中，并且在用户关闭浏览器时将被擦除，此类Cookie没有到期日期，并且不存储有关用户客户端设备的任何信息。它通常用于维护电子商务网站中的购物车内容。
  </p>
  <p class="graf graf--p">
   持久Cookie（例如 Flash 和 evercookie cookie）引起了严重的隐私问题。
  </p>
  <p class="graf graf--p">
   Cookie 的内容中有一半是第一方的，属于您访问的网站，另一半是第三方的，属于合作伙伴，服务或与网站合作的广告商。
  </p>
  <p class="graf graf--p">
   第三方 Cookie
   <strong class="markup--strong markup--p-strong">
    用于（跨多个网站的）跟踪活动
   </strong>
   ，并识别频繁访问和回访的访问者，以根据Cookie的历史记录量身定制内容或优化广告或改善用户体验。持久 cookie 的两种主要类型是 Flash 和 Evercookies。
  </p>
  <p class="graf graf--p">
   与具有有效日期并存储在客户端硬盘驱动器上特定文件夹中的传统cookie文件相比，Flash Cookies 具有更高的持久性。
   <strong class="markup--strong markup--p-strong">
    仅删除网络浏览器的 cookie 文件夹不会删除此类型的 cookie。
   </strong>
  </p>
  <p class="graf graf--p">
   Flash cookie 用于存储用户在多个网站上的浏览历史记录，可以用来重新实例化用户删除的 HTTP cookie。
  </p>
  <p class="graf graf--p">
   要访问存储在计算机中的所有 Flash cookie（在Windows下），请转到“控制面板”➤“ Flash Player”，然后选择“阻止所有站点在此计算机上存储信息”选项（参见下图）。
  </p>

   <strong class="markup--strong markup--p-strong">
    即使用户从其计算机中删除了 HTTP 和 Flash cookie 之后，它也可以存在。
   </strong>
  </p>
  <p class="graf graf--p">
   它通过将数据存储在客户端浏览器/计算机上的多个位置（例如，HTTP cookie，Flash cookie，HTML5本地存储，Web历史记录，Silverlight）来实现其持久性。
  </p>
  <p class="graf graf--p">
   例如，如果这些位置之一被用户删除，evercookie 将检测到该位置并自行重新生成。幸运的是，现代的网络浏览器能够阻止或检测 evercookies。
  </p>
  <p class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     3、ETag 追踪
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   ETag 是不使用 Cookie（HTTP和Flash）、JavaScript、HTML存储、或IP地址而跟踪用户的另一种方法。
  </p>
  <p class="graf graf--p">
   ETag或实体标签是超文本传输​​协议（HTTP）机制的一部分，该机制提供Web缓存验证，并用于控制特定文件在客户端缓存的时间。
  </p>
  <p class="graf graf--p">
   ETag帮助Web浏览器避免两次加载相同的Web资源，例如，当用户访问在后台播放音乐的网站时，它会根据用户的本地时间而改变。
  </p>
  <p class="graf graf--p">
   如果ETag不同，则客户端浏览器将下载新版本的音频文件。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    可以利用 ETag 以类似于持久cookie的方式跟踪用户
   </strong>
   ，并且，即使服务器上的内容不变，跟踪服务器也可以不断将ETag发送到客户端浏览器。这样，跟踪服务器可以与客户端计算机保持无限期持久的会话。
  </p>
  <p class="graf graf--p">
   要摆脱 ETag，必须清除浏览器缓存内容。
  </p>
  <p class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     4、数字指纹追踪
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   浏览器指纹是有关用户系统和浏览器的一组技术信息，可以在线区分其机器。
  </p>
  <p class="graf graf--p">
   此信息包括以下内容：浏览器类型、操作系统（OS）版本、安装的附件、用户代理、安装的字体、语言设置、时区、屏幕大小和颜色深度等。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    即使禁用了 cookie 和 JavaScript，指纹也可以使跟踪器区分用户的机器
   </strong>
   。
  </p>
  <p class="graf graf--p">
   浏览器指纹识别是无状态的，对用户和计算机都是透明的。
  </p>
  <p class="graf graf--p">
   从数字指纹收集的信息看似通用，不足以在线识别数百万个已连接设备中的单个计算机；但是，
   <strong class="markup--strong markup--p-strong">
    如果将这些信息组合在一起，则可以绘制有关每台用户计算机的全面而独特的图景，此后，如果将其与其他个人身份信息（PII）组合在一起，则可以将该信息追踪到真实身份
   </strong>
   。
  </p>
  <p class="graf graf--p">
   这应该有效地允许不同的外部方在不使用传统跟踪技术（例如计算机IP地址和cookie）的情况下轻松地对目标人进行描述。
  </p>

   <strong class="markup--strong markup--p-strong">

   </strong>
  </p>
  <p class="graf graf--p">
   设备指纹识别主要有两种类型：基于脚本的和基于画布的。
  </p>
  <p class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     基于脚本的指纹 ——
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   脚本指纹通过将脚本（通常为 JavaScript）加载到用户的浏览器中而起作用。成功加载脚本后，它将执行以提取有关当前浏览器和系统配置的大量技术信息。
  </p>
  <p class="graf graf--p">
   提取的信息包括用户代理、已安装的附件/扩展程序、已安装的字体、屏幕分辨率、时区、操作系统类型和版本、CPU和GPU类型、以及有关目标系统的许多其他详细信息。
  </p>
  <p class="graf graf--p">
   然后根据脚本收集的信息进行哈希处理。
   <strong class="markup--strong markup--p-strong">
    该哈希值可以识别和跟踪您的设备，就像IP地址一样。
   </strong>
  </p>
  <p class="graf graf--p">
   跟踪器使用 Flash、Silverlight 或 Java applet 代替 JavaScript 来执行指纹识别。他们都将返回相同的结果。
  </p>
  <p class="graf graf--p">
   抵制此技术的主要方法是在浏览器中禁用 JavaScript。但是，这种方法不切实际，并且可能导致破坏大量网站（大多数Web设计框架现在都基于 JavaScript 来提供功能）。
  </p>

     画布指纹 ——
    </strong>
   </span>
  </p>
  <p class="graf graf--p">
   Canvas 是最初由Apple开发的 HTML5 元素；它用于使用 JavaScript API 在网页上绘制图形（线条、形状、文本、图像）和动画（例如游戏和横幅广告）。广告客户可以利用画布功能识别和追踪用户。
  </p>
  <p class="graf graf--p">
   画布指纹追踪是一种跟踪用户在线活动的新方法。只需在用户的客户端浏览器上绘制不可见的图像即可。
  </p>
  <p class="graf graf--p">
   该图像对于每个用户而言都是不同的，一旦绘制在客户端浏览器上，它将收集有关用户浏览器和计算机的不同技术信息。然后根据已收集的信息进行哈希处理。
  </p>
  <p class="graf graf--p">
   该哈希值将在用户访问的所有网站上保持一致（哈希值是从画布数据生成的）；
   <strong class="markup--strong markup--p-strong">
    这将有效地记录用户的浏览历史记录
   </strong>
   。
  </p>
  <p class="graf graf--p">
   尽管不能单独使用从画布指纹中收集的信息来识别用户，但是，可以将此指纹与其他来源结合使用，以
   <strong class="markup--strong markup--p-strong">

   </strong>

   你一定注意到了，大多数网站都有 Facebook 的“ Like”和“ Share”按钮，这些按钮有助于在用户的 Facebook 新闻源上共享内容。但是，您应该知道的是，
   <strong class="markup--strong markup--p-strong">
    只要您访问了具有 Facebook “点赞”或“分享”按钮的网页，即使您没有单击该按钮，Facebook 也会记录此访问！
   </strong>
  </p>
  <p class="graf graf--p">
   Facebook 跟踪不会在此时停止，因为他们可以在用户不知情的情况下使用嵌入在“点赞”和“共享”按钮中的隐藏代码，
   <strong class="markup--strong markup--p-strong">
    在不同网站上跟踪 **非 Facebook** 用户，他们一直在这样做！
   </strong>
  </p>
  <p class="graf graf--p">
   Twitter 的“关注”按钮，在跟踪在线用户方面也起着相同的作用，就像 Facebook 的“点赞”和“共享”按钮一样。
  </p>

  <h3 class="graf graf--p">
   <span style="color: #ff6600;">
    <strong class="markup--strong markup--p-strong">
     逃避在线跟踪
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   正如我们一直以来不断介绍的安全知识中强调的那样，可以使用不同的技术在线跟踪用户，而对付这些技术需要正确地实施不同的工具和策略。
  </p>
  <p class="graf graf--p">
   为了防止在线跟踪，用户需要执行三个
   <strong class="markup--strong markup--p-strong">
    最基本的步骤
   </strong>
   ：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    使用可靠的VPN服务隐藏IP地址。
   </li>
   <li class="graf graf--li">
    关闭浏览器后，删除（或拒绝）cookie和Web浏览器缓存。
   </li>
   <li class="graf graf--li">
    防止数字指纹技术对计算机进行描述。
   </li>
  </ul>
  <p class="graf graf--p">
   这需要认真的技术技能，明确的防御意识。对抗跟踪和指纹识别很困难，并且
   <strong class="markup--strong markup--p-strong">
    没有
   </strong>
   100％保证的技术解决方案可以解决此问题。
  </p>
  <p class="graf graf--p">
   要知道，即使将Web浏览器配置为具有更好的隐私性并安装了许多附件之后，熟练的对手仍然可以在很大程度上识别您的数字指纹。
  </p>
  <p class="graf graf--p">
   对抗浏览器指纹识别的最佳技术解决方案是使您的浏览器看起来像大多数浏览器的指纹一样。
  </p>
  <p class="graf graf--p">
   为此，建议使用一个新安装的复制Web浏览器，该浏览器手动配置为具有增强的隐私设置，而无需安装任何附加组件。
   <strong class="markup--strong markup--p-strong">
    该浏览器应从同样全新安装的虚拟机（例如 Virtual Box ）上运行。
   </strong>

  </p>
