  <p class="graf graf--p">
   曾经有朋友询问如何构建一个追查不到所有者身份的网站。这需要涉及多个方面，域名注册方面的隐身只是之一，还包括，比如购买非免费软件、注册高级服务、为访问专有数据库付费等等，每一步都可以暴露您的身份。
  </p>
  <p class="graf graf--p">
   本文分为三个主要部分。
  </p>
  <p class="graf graf--p">
   首先，简要介绍一下假名和匿名之间的区别。
  </p>
  <p class="graf graf--p">
   其次，分步介绍如何获得一个假名域名，同时使任何人都难以确定您是谁。
  </p>
  <p class="graf graf--p">
   最后，简要介绍其他挑战，包括有关如何应对其中一些挑战的详细信息。
  </p>
  <p class="graf graf--p">
   这些步骤综合起来足以使一个人至少在一段时间内隐瞒身份，就算是最有能力和最有耐心的监视者也很难了解您是谁。同时，这也是一个警告性的故事，关于，追求 “完美” 匿名性也许需要您舍弃一些东西。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     假名、匿名、和可追踪性
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   假名和匿名是不同的。假名指的是您在现实中从未使用过的名字；匿名一般指的是不论如何都找不到对应的身份。这两者经常被混用。
  </p>
  <p class="graf graf--p">
   这些概念可以像数字安全领域中的任何其他事物一样变得复杂，但是最基本的定义通常是最有用的。
  </p>
  <p class="graf graf--p">
   最基本层面讲，使用假名就像是笔名，人们基本知道您的假名，而且不难通过假名知道您的真名；但是，要想匿名，您必须防止任何人将这个名字链接到您的真实身份。
  </p>
  <p class="graf graf--p">
   幸运的是，这些东西很少是非黑即白的，安全人员一般会谈论
   <strong class="markup--strong markup--p-strong">
    “可追踪性”
   </strong>
   。重要的是要认识到，匿名性（无论您选择叫它什么）都是灰色地带。
  </p>
  <p class="graf graf--p">
   👉更具体地说，您需要问的是一个切实的问题：“我需要防止谁将我的假名与真实身份联系起来？”
  </p>
  <p class="graf graf--p">
   如果答案是 “那个精通技术的亿万富翁间谍同时也是我邻居”，那么您将必须真正的匿名；如果答案是 “不很聪明并且不太在意的人”，那么一个听起来很酷的化名也许就可以解决问题。
  </p>
  <p class="graf graf--p">
   当然，
   <strong class="markup--strong markup--p-strong">
    这种可追踪性梯度并非仅限于域名注册 —— 它涉及各种在线和离线操作，包括选择电子邮件地址的简单过程
   </strong>
   。
  </p>
  <p class="graf graf--p">
   假设您的名字叫王大锤。如果您使用地址
   <a class="__cf_email__" data-cfemail="a0d7c1cec7c4c1c3c8d5c9e0c7cdc1c9cc8ec3cfcd" href="/cdn-cgi/l/email-protection">
    [email protected]
   </a>
   注册一个电子邮件帐户，然后将消息发送给任何知道您名字的人，那么他们一看就能知道是谁发送的。
  </p>
  <p class="graf graf--p">
   为了获得假名，您可以注册一个地址，例如
   <a class="__cf_email__" data-cfemail="5c2839373d37353d2e353a296f6e6f6b1c3b313d3530723f3331" href="/cdn-cgi/l/email-protection">
    [email protected]
   </a>
   。但是，如果您的邮件实际显示为
   <a class="__cf_email__" data-cfemail="9febfaf4fef4f6feedf6f9eaacadaca8dff8f2fef6f3b1fcf0f2" href="/cdn-cgi/l/email-protection">
    [email protected]
   </a>
   &lt; wangdachui&gt; 发送的邮件，那么显然，它根本就没什么用。
  </p>
  <p class="graf graf--p">
   同样，比如为 iyouport.org 购买域名，同时在 WHOIS 中公开列出
   <a class="__cf_email__" data-cfemail="85ecfceaf0f5eaf7f1c5e2e8e4ece9abe6eae8" href="/cdn-cgi/l/email-protection">
    [email protected]
   </a>
   作为注册人的电子邮件地址。从技术上讲，该域是伪名，但是，几乎不提供匿名性。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     公开的和私密的可追踪性
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   区分可追踪性是公开的还是私密的，也许很重要。
  </p>
  <p class="graf graf--p">
   <a class="__cf_email__" data-cfemail="c6b2a3ada7adafa7b4afa0b3f5f4f5f186a1aba7afaae8a5a9ab" href="/cdn-cgi/l/email-protection">
    [email protected]
   </a>
   &lt; wangdachui&gt; 是公开可追踪性的一个例子；如果在购买域名时，WHOIS 注册信息中包含您的真实联系信息，也是如此。任何知道该如何搜索的人都能拿到您参与其中的证据。
  </p>
  <p class="graf graf--p">
   私密可追踪性相比下更复杂一些。即使您在选择假名电子邮件地址的 &lt;Real Name&gt; 部分时非常小心 —— 或提交了精心选择的WHOIS注册信息 ——
   <strong class="markup--strong markup--p-strong">
    仍然会有许多个人、公司、机构和机器人能够将您的假名与您的真实身份联系在一起
   </strong>
   。
  </p>
  <p class="graf graf--p">
   👉比如下面这样的情况：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    这个电邮地址您曾经用过多年；
   </li>
   <li class="graf graf--li">
    您曾经将这个电邮地址链接到了个人社交媒体账户；
   </li>
   <li class="graf graf--li">
    您用信用卡付款了；
   </li>
   <li class="graf graf--li">
    创建帐户时提供了 “备用地址” 或手机号码；
   </li>
   <li class="graf graf--li">
    您经常从您的家和办公室的IP地址登录帐户 ……
   </li>
  </ul>
  <p class="graf graf--p">
   如果以上任何一个条件为 “是”，都能说明您的 “私密可追踪性” 很高。
  </p>
  <p class="graf graf--p">
   虽然私密可追踪性仅对那些占据某种特权的人而言是危险的，但是您需要清楚地列出这些人的名单，非常重要 —— ⚠️它们
   <strong class="markup--strong markup--p-strong">
    至少
   </strong>
   包括：熟悉您的所有人、域名注册商、电子邮件和社交网络服务提供商、ISP、任何监视机构、以及有机会物理访问您设备的任何人。
  </p>
  <p class="graf graf--p">
   👉通常最有可能的威胁包括：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    域名注册商或域名隐私保护服务提供商的员工收受贿赂而出卖您；
   </li>
   <li class="graf graf--li">
    信用卡公司出售交易信息以牟利的数据经纪人出卖您；
   </li>
   <li class="graf graf--li">
    经常与政府机构共享IP地址数据的互联网服务提供商（ISP）；
   </li>
   <li class="graf graf--li">
    服务提供商和手机运营商被政府间谍要求透露特定电话号码或电子邮件地址的所有者……
   </li>
  </ul>
  <p class="graf graf--p">
   ⚠️请注意，以上这些仅仅是最基本的！完整列表要长得多，它还可能包括您身边的所有人，朋友、知己、同事；而且，当然，还包括被黑客入侵并找到其收件箱、交易日志、和用户数据库并在线倾销以供所有人查看的情况，私密可追踪能力突然变成公开追踪性。
  </p>
  <p class="graf graf--p">
   因此，与其考虑有关假名性和匿名性的部分，不如考虑公开可追踪性部分和私密可追踪性部分。弄清面临的挑战，有助于帮您构思一个更合适的护盾。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     域名所有权和WHOIS
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   互联网名称与数字地址分配机构（ICANN）是于1998年成立的美国非营利组织。除其他职责外，它还管理分配域所有权的协议。出于本文的目的，
   <a class="markup--anchor markup--p-anchor" data-href="https://www.icann.org/resources/pages/ra-agreement-2009-05-21-en#3.7.7.2" href="https://www.icann.org/resources/pages/ra-agreement-2009-05-21-en#3.7.7.2" rel="noopener noreferrer" target="_blank">
    这些协议
   </a>
   最重要的方面是它们要求域所有者：
  </p>
  <p class="graf graf--p">
   <em class="markup--em markup--p-em">
    向注册服务商提供准确详细的联系方式 … 包括：全名、邮政地址、电子邮件地址、语音电话号码、和传真号码（如果有的话）
   </em>
  </p>
  <p class="graf graf--p">
   该协议进一步指出：
  </p>
  <p class="graf graf--p">
   <em class="markup--em markup--p-em">
    故意提供不准确或不可靠的信息 … 或 … 未能在15天内答复注册服务商有关联系方式的准确性询问 … 应作为取消注册资格的依据。
   </em>
  </p>
  <p class="graf graf--p">
   域名注册商通常将此联系信息提交到可公开访问的域名所有者的分布式数据库 WHOIS中。有许多免费的在线服务可让您搜索此数据库。而且非常简单，只需：
  </p>
  <ol class="postList">
   <li class="graf graf--li">
    在浏览器中导航到
    <a class="markup--anchor markup--li-anchor" data-href="https://whois.icann.org/" href="https://whois.icann.org/" rel="noopener noreferrer" target="_blank">
     ICANN WHOIS
    </a>
    页面
   </li>
   <li class="graf graf--li">
    输入一个域名
   </li>
   <li class="graf graf--li">
    点击查找
   </li>
  </ol>
  <h3 class="graf graf--p">
   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     如何获得一个匿名的域名
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   有两种方法可以做到这点。
  </p>
  <p class="graf graf--p">
   首先，付费购买域隐私保护服务。对于大多数人来说，这种方法可能就足够了，即使不试图隐藏您的身份，通常这样做也是个好主意。第二种方法要求违反您建立域 “所有权” 所依据的协议，但是会相对可靠。
  </p>
  <p class="graf graf--p">
   下面分别介绍这两种。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     域隐私保护服务
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   使用
   <a class="markup--anchor markup--p-anchor" data-href="https://en.wikipedia.org/wiki/Domain_privacy" href="https://en.wikipedia.org/wiki/Domain_privacy" rel="noopener noreferrer" target="_blank">
    域隐私保护服务
   </a>
   是最常见的方法。每月几欧元的费用，除了域名本身的费用外这些公司将收集并存储您的联系信息单独保留，而不是将其提交给 WHOIS。如果有人打电话问你是谁，他们不会把你说出来：
  </p>

   ，这其中写到他们将：

  <p class="graf graf--p">
   <em class="markup--em markup--p-em">
    合理及时地回应传票和其他法律程序 …… 满足寻求信息、文件或其他商业记录的请求，并根据适用法律和事实评估每个请求。每个请求将收取$ 75的管理费 ……
   </em>
  </p>
  <p class="graf graf--p">
   并且，如果您的帐户违反了其
   <a class="markup--anchor markup--p-anchor" data-href="http://perfectprivacy.com/aup.aspx" href="http://perfectprivacy.com/aup.aspx" rel="noopener noreferrer" target="_blank">
    可接受的使用政策
   </a>
   ，在这种情况下，它们 “将不会保护您的身份”。
  </p>
  <p class="graf graf--p">
   而有些公司则更加明确。例如，满足以下任何一项的情况是，他们将保留其权利：
  </p>
  <p class="graf graf--p">
   （i）向第三方透露您提供的联系信息…
  </p>
  <p class="graf graf--p">
   （ii）在公共 WHOIS 数据库中填写了注册人的姓名、主要邮寄地址、电子邮件地址和/或电话号码…
  </p>
  <p class="graf graf--p">
   （iii）终止您对私人注册服务的订阅 ……
  </p>
  <p class="graf graf--p">
   👉在以下情况下同样可能会暴露您的身份：
  </p>
  <p class="graf graf--p">
   （i）如果任何第三方声称该域名侵犯了其商标、商号或其他合法权利，不论该声称是否有效；
  </p>
  <p class="graf graf--p">
   （ii）遵守任何适用的法律、政府法规或要求、ICANN 政策或要求、法庭传票、法院命令、执法机构或政府机构的要求；
  </p>
  <p class="graf graf--p">
   （iii）如果任何第三方威胁要采取与域名直接或间接相关的法律诉讼 … 或声称您以违反任何法律法规的方式使用域名注册，或以其他方式违法或侵犯了第三方的合法权利，无论此类主张是否有效。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    显然，您非常可能没有任何安全感，特别是在暴露后果严重的情况下。因为只要政府间谍或恶意第三方声称您侵权，就可以曝光您的身份
   </strong>
   。
  </p>
  <p class="graf graf--p">
   如上解释，域隐私保护服务仅仅提供了公开的不可追踪性，但是，很少能提供私密不可追踪性。
  </p>
  <p class="graf graf--p">
   他们提供的匿名性几乎完全取决于可能为您选择的某些公司本身的可靠性和可信赖性。
  </p>
  <p class="graf graf--p">
   尽管有时您可以自由地将业务交给自己感觉靠谱的ISP或注册商，但是，您几乎不可能了解这些域隐私服务的道德特征和运营安全性。
  </p>
  <p class="graf graf--p">
   那么，还剩下另一种方法，使用假名注册域名。
  </p>
  <h3 class="graf graf--p">
   <span style="color: #00ccff;">
    <strong class="markup--strong markup--p-strong">
     用假名注册域名
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">

  </p>
  <p class="graf graf--p">
   但是，这样做违反了您与注册服务商达成的 ICANN 授权协议的条款，因此
   <strong class="markup--strong markup--p-strong">
    您需要创建一个内部一致的假名
   </strong>
   。至少两点很重要：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    使用以现金购买的并以新名字和地址注册的预付费“礼品卡”来支付域名；
   </li>
   <li class="graf graf--li">

   <span style="color: #339966;">
    <strong class="markup--strong markup--p-strong">
     获取一个 “真的” 匿名域名
    </strong>
   </span>
  </h3>
  <p class="graf graf--p">
   部分原因是大多数域注册商都需要一个经过验证的电子邮件地址，因此通过执行上述步骤可以实现的匿名性有限，这些匿名性无法解决一些特定的私密可追踪性风险。
  </p>
  <p class="graf graf--p">
   👉在用于其他活动的电子邮件地址下注册将意味着允许某人观察这些活动与新域名之间的联系。风险取决于您公开使用该地址的方式、以及在其他方式上的谨慎程度 —— “某人” 可能包括您的电子邮件提供商、注册商、或ISP，以及其他可能感兴趣的各方。
  </p>
  <p class="graf graf--p">
   因此，以下步骤建议
   <strong class="markup--strong markup--p-strong">
    一种注册新电子邮件帐户的方法，不会在该地址和真实身份之间建立任何明显的联系
   </strong>
   。
  </p>
  <p class="graf graf--p">
   但是，由于现在许多电子邮件提供商都要求提供手机号码，因此您还必须考虑限制手机号码的可追踪性。
  </p>

   您可以通过以下步骤做到这点。

   </li>
   <li class="graf graf--li">
    使用该号码限制电子邮件地址的可追踪性
   </li>
   <li class="graf graf--li">
    如上所述，在 “以假名注册” 之前执行所有这些操作。
   </li>
  </ul>
  <h3 class="graf graf--p">
   <span style="color: #00ccff;">

    </strong>

   </li>
   <li class="graf graf--li">
    访问预付礼品卡帐户的时候
   </li>
   <li class="graf graf--li">
    使用预付礼品卡购物的时候
   </li>
   <li class="graf graf--li">
    访问、管理或搜索新域名信息的时候
   </li>
  </ul>

   坚持上述所有比听起来要难。但可以将所有相关密码短语保留在仅存在于 Tails USB 记忆棒上的 KeePassX 数据库中。
