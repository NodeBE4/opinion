  <h4 class="graf graf--h4">
   使用元数据
  </h4>
  <p class="graf graf--p">
   为了更好地理解如何使用元数据和围绕元数据，重要的是要在实践中知道提到元数据时通常意味着什么。
  </p>
  <p class="graf graf--p">
   下面是可以与不同类型的数据一起存储的元数据列表：
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    图片
   </strong>
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    如果使用 GPS 功能设备（如智能手机）拍摄照片的位置（纬度和经度坐标）；
   </li>
   <li class="graf graf--li">
    相机设置，如 ISO 感光度、快门速度、焦距、光圈、白平衡、镜头类型等（请注意，
    <strong class="markup--strong markup--li-strong">
     有些相机确实包含位置坐标
    </strong>
    ）；
   </li>
   <li class="graf graf--li">
    相机或智能手机的型号；
   </li>
   <li class="graf graf--li">
    拍摄照片的日期和时间；
   </li>
   <li class="graf graf--li">
    用于编辑照片的程序的名称；
   </li>
  </ul>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    PDF 文件
   </strong>
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    作者姓名，通常是首次安装用于创建文件的程序时分配的名称；
   </li>
   <li class="graf graf--li">
    用于创建文件的程序的版本和名称；
   </li>
   <li class="graf graf--li">
    文件的标题；
   </li>
   <li class="graf graf--li">
    某些关键字；
   </li>
   <li class="graf graf--li">
    文件创建/上次修改的日期和时间；
   </li>
   <li class="graf graf--li">
    文本文件；
   </li>
  </ul>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    根据用于创建文档的程序，数据可能包括
   </strong>
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    所有不同作者的名字；
   </li>
   <li class="graf graf--li">
    在以前版本的文档中删除的文本和注释行；
   </li>
   <li class="graf graf--li">
    创建和修改日期；
   </li>
  </ul>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    影片
   </strong>
  </p>
  <p class="graf graf--p">
   视频文件中的元数据可分为两部分：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    自动生成的元数据：创建日期，大小，格式，编解码器，持续时间，位置；
   </li>
   <li class="graf graf--li">
    手动添加的元数据：有关素材的信息，文本转录，标签，更多信息以及编辑的注释等等；
   </li>
  </ul>
  <p class="graf graf--p">
   *推荐阅读：全面了解视频元数据并与
   <a class="markup--anchor markup--p-anchor" data-href="https://blog.witness.org/2014/10/video-metadata/" href="https://blog.witness.org/2014/10/video-metadata/" rel="noopener noreferrer" target="_blank">
    WITNESS 合作
   </a>
   。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    音频
   </strong>
  </p>
  <p class="graf graf--p">
   音频元数据类似于视频，但更广泛地用于注册文件的属性。除此之外，还可以包括：创建日期，大小，格式，编解码器，持续时间和一组手动添加的数据，如标签，艺术家信息，艺术作品，评论，专辑中的曲目编号，流派等等。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    通讯
   </strong>
  </p>
  <p class="graf graf--p">
   通信中的元数据取决于所使用的通信类型（即电子邮件，移动电话，智能手机……等）。但总的来说，它可以揭示以下内容（如果没有使用隐藏元数据的工具的话）：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    发送者和接收者的 ID；
   </li>
   <li class="graf graf--li">
    沟通的日期和时间；
   </li>
   <li class="graf graf--li">
    地点；
   </li>
   <li class="graf graf--li">
    沟通方式..等等；
   </li>
  </ul>
  <h4 class="graf graf--h4">
   工具
  </h4>
  <p class="graf graf--p">
   有多种方法可以从文件中提取元数据。选项因操作系统而异，从工具到插件，再到桌面版本或浏览器内工具。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    免责声明
   </strong>
   ：⚠️ 使用在线平台提取元数据时，牢记数字隐私和安全非常重要。没有足够的信息来保证流程的机密性。这些平台可能会跟踪您的在线行为，存储您的数据或与第三方或当局共享。⚠️
  </p>
  <p class="graf graf--p">
   有各种方法可以揭示或查看元数据，这些方法和工具将在后面详细介绍。有些工具可以读取内置文件信息的元数据（例如 Photoshop），这意味着它们将以其格式显示数据。其他的则有更详细的输出。
  </p>

   元数据工具

  </ul>
  <p class="graf graf--p">
   这个我们已经多次提到了。CameraV 是由卫报项目和 WITNESS 创建的移动应用程序。应用程序名称中的 V 代表验证，它是为了向照片或视频添加大量额外元数据而创建的，以验证其真实性。这个软件做了两件事。首先，它描述了图像和视频的人物，内容，时间，地点，原因和方式。其次，它建立了一个可以在法庭上指出的监管链。对于公民记者取证来说，非常好用。
  </p>
  <p class="graf graf--p">
   <a class="markup--anchor markup--p-anchor" data-href="https://exposingtheinvisible.org/resources/obtaining-evidence/image-digging" href="https://exposingtheinvisible.org/resources/obtaining-evidence/image-digging" rel="noopener noreferrer" target="_blank">
    Metadata En Masse
   </a>
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    兼容性：Linux，Mac OS
   </li>
   <li class="graf graf--li">
    专有状态：免费和开源
   </li>
   <li class="graf graf--li">
    类型：桌面
   </li>
  </ul>
  <p class="graf graf--p">
   正如其名字所示，该脚本允许从大量图像中提取地理定位元数据。在处理大量图像时，它可以节省宝贵的时间。由 Exposing the Invisible 团队成员编写的脚本应放在名为 geobatch.rb 的文件中，并在包含其中所有图像的文件夹中运行。
  </p>
  <p class="graf graf--p">
   <a class="markup--anchor markup--p-anchor" data-href="https://itunes.apple.com/us/app/trashexif-metadata-photo-remover/id585543219" href="https://itunes.apple.com/us/app/trashexif-metadata-photo-remover/id585543219" rel="noopener noreferrer" target="_blank">
    <strong class="markup--strong markup--p-strong">
     TrashEXIF
    </strong>
   </a>
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    兼容性：Mac OS
   </li>
   <li class="graf graf--li">
    类型：手机
   </li>
  </ul>
  <p class="graf graf--p">
   TrashEXIF 是一款 iPhone 应用程序，允许用户从图像中删除所有元数据或控制应删除或保留哪些元数据。该应用程序还允许预设应用于所有拍摄图像的协议。
  </p>
  <h4 class="graf graf--h4">
   删除元数据
  </h4>
  <p class="graf graf--p">
   有多种方法可以从文件中删除元数据。以下是来自
   <a class="markup--anchor markup--p-anchor" data-href="https://securityinabox.org/" href="https://securityinabox.org/" rel="noopener noreferrer" target="_blank">
    Security in-a-Box
   </a>
   工具包的一些建议。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    先发制人
   </strong>
  </p>
  <p class="graf graf--p">
   您可以通过以下方式阻止捕获 GPS 位置等特定类型的元数据：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    关闭无线和 GPS 位置（位置服务下）和移动数据（可以在数据管理器 →数据传输下找到）；
   </li>
   <li class="graf graf--li">
    拍摄照片时，请确保照片应用程序中标记位置的设置也已关闭；
   </li>
   <li class="graf graf--li">
    使用 Metanull（适用于 Windows）等工具，您可以确保在共享之前删除所有元数据。下面详细讨论该工具。
   </li>
  </ul>
  <p class="graf graf--p">
   注意：⚠️ 某些文件（如 DOC 和 PDF）可以在其中保存图像文件。如果您没有采取必要的谨慎措施，只是清除了保存图像的文档上的元数据，但是将依旧保留被嵌入图像的元数据！在将图像添加到 DOC 之前请使用 Metanull 事先从中删除所有元数据。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    从文档和其他文件中删除元数据
   </strong>
  </p>
  <p class="graf graf--p">
   如上所述，其他常用文件类型（如可移植文档文件（PDF）或由 Microsoft Office 或 LibreOffice 等应用程序创建的文字处理文档）包含元数据，其中可能包括：
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    创建文档的人员的用户名；
   </li>
   <li class="graf graf--li">
    最近编辑和保存文档的人员姓名；
   </li>
   <li class="graf graf--li">
    创建和修改文档的日期。
   </li>
  </ul>
  <p class="graf graf--p">
   在某些情况下，
   <strong class="markup--strong markup--p-strong">
    您的文档可能还包含其他个人身份信息，例如地址，电子邮件地址，政府 ID，IP 地址或与您计算机上其他程序中的个人身份信息相关的唯一标识符
   </strong>
   。
  </p>
  <p class="graf graf--p">
   通过查看文件属性（可以通过右键单击文件图标并选择‘属性’来访问），可以轻松访问其中一些信息。查看其他信息或隐藏性数据则需要特定的软件。在任何情况下，根据环境的不同而异，如果您正在交换敏感信息，此数据就可能会使您面临风险。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    从 PDF 文件中删除元数据
   </strong>
  </p>
  <p class="graf graf--p">
   Windows 或 MAC OS 用户可以使用 Adobe Acrobat XI Pro（提供试用版）等程序从 PDF 文件中删除或编辑隐藏数据。
  </p>
  <p class="graf graf--p">
   使用 Acrobat 打开任何 PDF 文件将允许您通过转到“文件”菜单然后选择属性来编辑元数据。在这里，您可以修改文档作者的姓名、标题、主题、关键字和任何其他元数据。您可以通过转到“工具”菜单，然后选择“保护”，再选择“删除隐藏的信息”，从而删除有关创建时间、修改时间、用于创建文件的设备类型以及您未看到的其他隐藏性数据的信息。
  </p>
  <p class="graf graf--p">
   对于 GNU / Linux 用户，PDF MOD 是一个免费的开源工具，用于编辑和删除 PDF 文件中的元数据。但是，它不会删除创建或修改时间，也不会删除用于创建 PDF 的设备类型。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    从 LibreOffice 文档中删除元数据
   </strong>
  </p>
  <p class="graf graf--p">
   在 LibreOffice 文档中，可以通过选择“文件”菜单，然后选择“属性”来查看元数据。在“常规”选项卡下，可以单击“重置”以重置常规用户数据，例如总编辑时间和修订号。您还可以通过取消选中此屏幕上的“应用用户数据”复选框，以便删除创建者的名称。完成后，转到“描述”和“自定义属性”选项卡以清除您不想显示的任何数据。最后，单击 Securit **选项卡并取消选中*Record 更改框，如果默认情况下未取消选中它的话。
  </p>
  <p class="graf graf--p">
   注意：如果使用 Versions 功能，则可以通过转到“文件”菜单和“版本”来删除可能存储在其中的旧版本文档。如果您使用“更改”功能，请转到“编辑”菜单，然后单击“更改”以接受或拒绝清除与文档所做更改相关的数据（如果您不再需要此信息的话）。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    用于清理元数据的其他策略
   </strong>
  </p>
  <p class="graf graf--p">
   某些文件类型包含的元数据多于其他文件类型，因此如果您不想使用软件，并且文件的格式无关紧要，您可以从包含大量元数据的文件中做出更改（例如 .DOCs 和 .JPEGs）变成（例如 .TXTs 和 .PNGs）。
  </p>
  <p class="graf graf--p">
   ⚠️ 在注册 Microsoft Office，Open Office，Libre Office，Adobe Acrobat 等软件的副本时，请避免使用您的真实姓名、地址、公司或组织名称。如果您必须提供姓名或地址，请使用假名。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
