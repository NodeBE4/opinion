   <li>
    <span style="color: #00ccff;">
     <em>
      我们都认为更新是为了更安全；但是 facebook 的更新是为了更好地监视你。曝光的内部谈话记录揭示了该巨头令人震惊的事实。是时候认清真相了
     </em>
    </span>
   </li>
  </ul>
  <p class="graf graf--p graf-after--figure" id="932d">
   又是新的一周，又是一系列事件提醒我们，尽管 Facebook 乐于标榜自己为一家“乐观的”公司，仅仅单纯地帮助用户联络全世界，但现实情况则完全不同。本周的这些提醒信息包括一系列新曝光的文件，这些文件建议公司在明知道会损害用户和破坏创新的情况下实现某些特性和规章。
  </p>
  <p class="graf graf--p graf-after--p" id="fe18">
   上周，一位英国议会成员公布了来自 Facebook 的
   <a class="markup--anchor markup--p-anchor" data-href="https://www.parliament.uk/documents/commons-committees/culture-media-and-sport/Note-by-Chair-and-selected-documents-ordered-from-Six4Three.pdf" href="https://www.parliament.uk/documents/commons-committees/culture-media-and-sport/Note-by-Chair-and-selected-documents-ordered-from-Six4Three.pdf" rel="noopener nofollow noreferrer" target="_blank">
    一批文件
   </a>
   其来源为一个叫做 Six4Three 的公司的部分法律诉讼程序。这些电子邮件，备忘录，以及幻灯片曝光了 Facebook 在
   <a class="markup--anchor markup--p-anchor" data-href="https://www.eff.org/deeplinks/2018/03/how-change-your-facebook-settings-opt-out-platform-api-sharing" href="https://www.eff.org/deeplinks/2018/03/how-change-your-facebook-settings-opt-out-platform-api-sharing" rel="noopener nofollow noreferrer" target="_blank">
    Cambridge Analytica 丑闻
   </a>
   事件发生之前、期间及之后发生的不为人知的事。
  </p>
  <p class="graf graf--p graf-after--p" id="7ed8">
   以下内容是这250页左右的文件中的关键部分。
  </p>
  <p class="graf graf--p graf-after--p" id="dc14">
   <strong class="markup--strong markup--p-strong">
    ⚠️ Facebook 利用新的 Android 更新以更加恐怖的方式窥探你的生活隐私
   </strong>
  </p>
  <p class="graf graf--p graf-after--p" id="9370">
   此文件包括一些内部的讨论，通过这些讨论可以看到
   <a class="markup--anchor markup--p-anchor" data-href="https://www.eff.org/deeplinks/2018/03/android-users-change-setting-stop-facebooks-collection-your-call-and-text-metadata" href="https://www.eff.org/deeplinks/2018/03/android-users-change-setting-stop-facebooks-collection-your-call-and-text-metadata" rel="noopener nofollow noreferrer" target="_blank">
    Facebook Messenger 暗中记录 Android 用户语音通话和文字信息历史的行为
   </a>
   。今年春天某位用户
   <a class="markup--anchor markup--p-anchor" data-href="https://arstechnica.com/information-technology/2018/03/facebook-scraped-call-text-message-data-for-years-from-android-phones/" href="https://arstechnica.com/information-technology/2018/03/facebook-scraped-call-text-message-data-for-years-from-android-phones/" rel="noopener nofollow noreferrer" target="_blank">
    发现了 Messenger 的这种行为
   </a>
   。此事件紧随 Cambridge Analytica 新闻，导致了公众的愤怒。Facebook 以“真实性调查” 的
   <a class="markup--anchor markup--p-anchor" data-href="https://newsroom.fb.com/news/2018/03/fact-check-your-call-and-sms-history/" href="https://newsroom.fb.com/news/2018/03/fact-check-your-call-and-sms-history/" rel="noopener nofollow noreferrer" target="_blank">
    新闻稿件
   </a>
   进行回应，坚称 Messenger 从未在未取得用户授权的情况下收集此类数据。
  </p>
  <p class="graf graf--p graf-after--p" id="d39d">
   <a class="markup--anchor markup--p-anchor" data-href="https://twitter.com/ashk4n/status/1070378861047635968" href="https://twitter.com/ashk4n/status/1070378861047635968" rel="noopener nofollow noreferrer" target="_blank">
    然而
   </a>
   ，根据最新泄漏的 2015 年的文件，Facebook 雇员讨论了强制用户升级到一个新的、更容易被窃取隐私的 Messenger 版本而“完全不会触发 Android 的权限对话框“的计划，尽管他们意识到这类滥用应用程序功能的行为“从 PR 的角度来看是一个风险及高的事情”。
  </p>

    </p>

    </p>

    </p>

    <p class="graf graf--p graf-after--p" id="d07e">

    </p>

    </p>
    <p class="graf graf--p graf--startsWithSingleQuote graf-after--p" id="9eb8">
     ‘这仍将是一个突破性的更改，所以用户将不得不点击升级，
    </p>
    <p class="graf graf--p graf-after--p" id="9ea9">

     <a class="markup--anchor markup--p-anchor" data-href="https://mislove.org/publications/PII-PETS.pdf" href="https://mislove.org/publications/PII-PETS.pdf" rel="noopener nofollow noreferrer" target="_blank">

     电话号码作为广告投放目标的研究和
     <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/facebook-is-giving-advertisers-access-to-your-shadow-co-1828476051" href="https://gizmodo.com/facebook-is-giving-advertisers-access-to-your-shadow-co-1828476051" rel="noopener nofollow noreferrer" target="_blank">
      调查
     </a>
     。令人不安的是其中提到将电话和文字消息历史记录提供给可怕的臭名昭著的 PYMK — — 或称之为“你可能知道的人(People You May Know)“，用于好友推荐。
    </p>
    <p class="graf graf--p graf-after--p" id="a2a6">
     <strong class="markup--strong markup--p-strong">
      ⚠️ “我想我们是把数据泄露给了开发者”
     </strong>
    </p>
    <p class="graf graf--p graf-after--p" id="7783">
     这些文件的中心主题是关于 Facebook 应该如何选择以使其他开发着使用其用户数据。他们透露，马克·扎克伯格很早就已认可访问 Facebook 的数据对其他公司来说是极其
    </p>
    <p class="graf graf--p graf-after--p" id="b37a">
     有价值的，而且 Facebook 领导层决心权衡其价值。
    </p>
    <p class="graf graf--p graf-after--p" id="9a3f">
     一些上下文解释：2010年，Facebook 启用了图形应用程序接口 1.0 版，这是一套非常强大且包容的工具，第三方开发者可以利用此工具访问有关应用程序用户及其好友的数据。
    </p>
    <p class="graf graf--p graf-after--p" id="e472">
     数十封电子邮件告诉我们公司是如何讨论利用这些数据的访问获利的。公司决策者准备了几种不同的方案，从向特定开发者按用户访问量收费，到
     <strong class="markup--strong markup--p-strong">
      要求“[Facebook] 不想与其分享数据”的应用程序每年在 Facebook 的广告平台上花费特定的钱数，否则将失去他们的数据
     </strong>
     。
    </p>
