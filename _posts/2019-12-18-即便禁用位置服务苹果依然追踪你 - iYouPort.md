  <p class="graf graf--p">
   苹果宣称为隐私服务 —— 给您提供最好的隐私保护，但是随后，一份报告发现，最新一代的 iPhone 可能无法像苹果用户所希望的那样让用户对其隐私进行完全的控制。
  </p>
  <p class="graf graf--p">
   来自 KrebsOnSecurity 的 Brian Krebs
   <a class="markup--anchor markup--p-anchor" data-href="https://krebsonsecurity.com/2019/12/the-iphone-11-pros-location-data-puzzler/" href="https://krebsonsecurity.com/2019/12/the-iphone-11-pros-location-data-puzzler/" rel="noopener noreferrer" target="_blank">
    一份报告
   </a>
   发现，尽管在隐私政策中告诉用户他们可以 “随时禁用位置服务”，但是即使已经手动关闭所有设置和应用程序的位置权限，Apple 本身也会继续追踪用户的位置。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    不要错过《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%e5%8d%b3%e4%bd%bf%e6%82%a8%e6%8b%92%e7%bb%9d%e6%8e%88%e4%ba%88%e6%9d%83%e9%99%90%ef%bc%8c%e4%bb%8d%e6%9c%891000%e5%a4%9a%e4%b8%aaandroid%e5%ba%94%e7%94%a8%e7%bb%a7%e7%bb%ad%e6%94%b6%e9%9b%86/" href="https://www.iyouport.org/%e5%8d%b3%e4%bd%bf%e6%82%a8%e6%8b%92%e7%bb%9d%e6%8e%88%e4%ba%88%e6%9d%83%e9%99%90%ef%bc%8c%e4%bb%8d%e6%9c%891000%e5%a4%9a%e4%b8%aaandroid%e5%ba%94%e7%94%a8%e7%bb%a7%e7%bb%ad%e6%94%b6%e9%9b%86/" rel="noopener noreferrer" target="_blank">
     即使您拒绝授予权限，仍有1,000多个Android应用继续收集你的私密数据
    </a>
    》
   </li>
  </ul>
  <p class="graf graf--p">
   Krebs 在运行最新版本 iOS 13 的新 iPhone 11 Pro 上发现了此问题，尽管之前
   <a class="markup--anchor markup--p-anchor" data-href="https://discussions.apple.com/thread/250665845" href="https://discussions.apple.com/thread/250665845" rel="noopener noreferrer" target="_blank">
    其他
   </a>
   用户
   <a class="markup--anchor markup--p-anchor" data-href="https://discussions.apple.com/thread/250880651" href="https://discussions.apple.com/thread/250880651" rel="noopener noreferrer" target="_blank">
    已经
   </a>
   在Apple论坛上报告了始终在线位置跟踪的问题。
  </p>
  <p class="graf graf--p">
   在该问题的演示视频中，Krebs 从 “设置” 中的 “位置服务” 菜单手动禁用了“地图”、“ Siri” 和各个 “系统服务” 设置，只是启用了箭头状的状态栏图标。
  </p>
  <p class="graf graf--p">
   然后，Krebs 导航到控制中心，并将手机置于飞行模式。当 Krebs 关闭飞行模式时，尽管设置之外没有打开任何应用程序，但状态栏中仍显示箭头图标。
  </p>
  <p class="graf graf--p">
   <iframe allowfullscreen="allowfullscreen" height="421" src="//www.youtube.com/embed/37_3hd_SK24" width="750">
   </iframe>
  </p>
  <p class="graf graf--p">
   Krebs 表示，尽管同样运行的是最新版本的iOS 13，但他无法在iPhone 8上重复该问题，并推测该问题可能与iPhone 11的硬件和对WiFi 6的支持有关。
  </p>
  <p class="graf graf--p">
   苹果公司的隐私政策最近一次更新是在 iPhone 11 发布之前的八月份，它指出苹果 “可能会收集、使用和共享精确的位置数据，包括苹果计算机或设备的实时地理位置。”
  </p>
  <p class="graf graf--p">
   但是正如 Krebs 所指出的那样，位置服务中的设备上隐私政策似乎暗示可以完全关闭此数据收集。
  </p>
  <p class="graf graf--p">
   隐私权政策指出：“您可以随时禁用位置服务。” “您还可以通过关闭每个基于位置的服务来禁用基于位置的系统服务。”
  </p>
  <p class="graf graf--p">
   但是，在 Krebs 与该公司联系后，一名苹果工程师表示，定位图标只对于未在“设置”中进行切换的系统服务显示。也就是说这表明用户实际上不能禁用所有基于位置的系统服务，这恰恰是Apple的隐私权政策所建议的。
  </p>
  <p class="graf graf--p">
   考虑到对隐私的卓越表现是该公司
   <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/apples-newest-luxury-product-is-privacy-1835233518" href="https://gizmodo.com/apples-newest-luxury-product-is-privacy-1835233518" rel="noopener noreferrer" target="_blank">
    的骄傲
   </a>
   ，苹果设备即使在用户采取措施阻止位置跟踪后仍可以继续跟踪其位置，这尤其令人好奇。
  </p>
  <p class="graf graf--p">
   该公司在去年的CES上大胆地摆出了
   <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/apples-newest-luxury-product-is-privacy-1835233518" href="https://gizmodo.com/apples-newest-luxury-product-is-privacy-1835233518" rel="noopener noreferrer" target="_blank">
    巨大的广告牌
   </a>
   ，上面写着 “happens on your iPhone, stays on your iPhone.” 但是，如果用户甚至没有控制自己的 iPhone 的功能以阻止公司跟踪其位置的权力，那么事实显然并非如此。
  </p>
  <p class="graf graf--p">
   尽管此问题似乎与苹果自己的隐私权政策相抵触，但这也不是该公司第一次误导用户有关其iPhone上的位置跟踪的信息。
  </p>
  <p class="graf graf--p">
   早在2017年就有报道浮出水面，涉及 iOS 11 中的控制中心，尽管看起来已经关闭，但 iPhone
   <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/the-ios-11-control-center-misleads-users-on-whether-the-1818604079" href="https://gizmodo.com/the-ios-11-control-center-misleads-users-on-whether-the-1818604079" rel="noopener noreferrer" target="_blank">
    实际上并未
   </a>
   与 wifi 和蓝牙断开连接。
  </p>
  <p class="graf graf--p">
   正如主板当时指出的那样，尽管苹果确实在其支持页面上对此做出了回应，但改变对用户来说并不明显。 （苹果后来推出了
   <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/apple-is-testing-a-beta-fix-to-the-ios-11-control-cente-1820420503" href="https://gizmodo.com/apple-is-testing-a-beta-fix-to-the-ios-11-control-cente-1820420503" rel="noopener noreferrer" target="_blank">
    修复程序
   </a>
   ）
  </p>
  <p class="graf graf--p">
   尽管 Apple 工程师在给 Krebs 的评论指出，“设置” 中没有打开开关的系统服务似乎显示了箭头图标，但这似乎是一个bug。
  </p>
  <p class="graf graf--p">
   苹果公司最终解释说，“这是用于合规性的设备上流程的一部分”，它承诺会提供一种新工具以便让用户能在将来关闭该功能。您可以
   <a class="markup--anchor markup--p-anchor" data-href="https://gizmodo.com/apple-explains-whats-going-on-with-its-big-location-ser-1840244321" href="https://gizmodo.com/apple-explains-whats-going-on-with-its-big-location-ser-1840244321" rel="noopener noreferrer" target="_blank">
    在此处
   </a>
   阅读完整的说明。
  </p>
  <p class="graf graf--p">
   也就是说，目前仍在跟踪。⚪️
  </p>
