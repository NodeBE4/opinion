       欧盟邪恶的版权法案的签署被全世界视为互联网最黑暗的一天。但是事实上它已经过时了，只需要一个条件，是什么？
      </strong>

    </span>
   </li>

  </figure>
  <p>
   我们生活在一个由各国分散立法监管的中央网络中。因此，如果我在荷兰建立一个 Web 平台，我就需要考虑平台上提供的内容在德国是否合法（这里可以替换为任何国家），我的用户也可以来自任何国家。
  </p>
  <p>
   现行立法的这一限制和其他限制阻碍了欧洲企业的经济潜力。因此，2014年，欧洲委员会（EC）在数字单一市场（DSM）战略下为欧洲提出了令人钦佩的工作愿景：
  </p>
  <p>
   <em>
    “互联网和数字技术正在改变我们的世界[…]互联网公司、初创公司的视野有限[…]现在是时候让欧盟的单一市场适应数字时代 — 摧毁监管墙[…]这可以为我们的经济每年贡献4150亿欧元，并创造数十万个新工作岗位。
   </em>
   “
  </p>
  <p>
   然而，根据最近关于版权改革最臭名昭著的第13条的提议，欧盟委员会正在将版权放在首位，并将市场限制在一定程度上，这可能会破坏上述愿景中的大部分。权利持有者不再需要指控某人侵权并证明这一点，现在平台提供商有责任表明他们没有侵权并为此目的尽一切努力。
  </p>
  <p>
   从本质上讲，EC 建议让平台提供商监管用户在其平台上发布的内容。换句话说，像Facebook，Twitter 和其他平台的提供商
   <strong>
    将不得不开始审查其用户发布的内容，以确保他们没有所谓的“侵犯版权”。
   </strong>
  </p>
  <p>
   最重要的是，他们有义务为他们允许的那些帖子寻求许可证（这对于如此众多的自制创意作品来说几乎是不可能的）。
  </p>
  <p>
   许多人撰写了大量详细的文章，讨论了拟议的平台过滤器的问题以及它与其他欧洲立法的矛盾或破坏性。其中包括 Max Planck Society 报告，EDRi 第13条解构，以及由57个相当著名的签署者签署的信函。
  </p>
  <p>
   他们展示了相当多的人和组织明确反对这一提议，因为如果不从根本上改变我们所知的网络，它将无法运作。
  </p>
  <figure class="wp-block-image">
   <img alt="" class="wp-image-3267 jetpack-lazy-image" data-lazy-sizes="(max-width: 910px) 100vw, 910px" data-lazy-src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=910%2C534&amp;is-pending-load=1#038;ssl=1" data-lazy-srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?w=910&amp;ssl=1 910w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=300%2C176&amp;ssl=1 300w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=768%2C451&amp;ssl=1 768w" data-recalc-dims="1" height="534" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=910%2C534&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="910"/>
   <noscript>
    <img alt="" class="wp-image-3267" data-recalc-dims="1" height="534" sizes="(max-width: 910px) 100vw, 910px" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=910%2C534&amp;ssl=1" srcset="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?w=910&amp;ssl=1 910w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=300%2C176&amp;ssl=1 300w, https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-228.png?resize=768%2C451&amp;ssl=1 768w" width="910"/>
   </noscript>
  </figure>
  <p class="graf graf--p">
   我喜欢张贴 meme，比如上面这样，但是平台会被该法案激励去除这张图片，因为它可能会侵犯指环王的制作公司的版权。第13条可能使我们对网络的热爱变得非常困难。
  </p>
  <p class="graf graf--p">
   于是，
   <strong class="markup--strong markup--p-strong">
    分散化网络是未来的趋势这一结论已经变得越来越合理了
   </strong>
   ：万维网联盟已经发布了分散式社交网络的规范；像 Dat，IPFS 和 Secure Scuttlebutt 这样的协议正在成熟，Firefox 开放了它们使用这些协议的扩展，
   <strong class="markup--strong markup--p-strong">
    Brave
   </strong>
   浏览器正在实现其中的一些协议，并且
   <strong class="markup--strong markup--p-strong">
    有一个完整的浏览器致力于在这些协议之上构建分散的 Web ：就是 Beaker Browser
   </strong>
   。这些只是分散网络的众多计划中的一小部分。
  </p>
  <p class="graf graf--p">
   根据分散化网络的发展方式，欧盟的立法可能已经过时了。今天，
   <strong class="markup--strong markup--p-strong">
    分散式网络还只适用于早期采用者，但它可以很快成为主流，因为它有助于解决人们现在关注的重点问题
   </strong>
   — — 它可以减少垃圾邮件，提高安全性，简化身份验证，提高可靠性，并使 Web 应用程序也可以更容易脱机工作。
  </p>
  <p class="graf graf--p">
   请注意，这不是一个思想实验，
   <strong class="markup--strong markup--p-strong">
    分散化 Web 现在已经存在。以 Fritter 和 Rotonde 为例，这是两个分散化的社交微博应用程序
   </strong>
   。当我在那里发布内容时，我不会将其发布在平台上 — — 我将其发布在我的计算机上并直接提供给其他人。就像 Kit Kat 下面GIF中所做的那样。
  </p>
  <figure class="wp-block-image">
   <img alt="" class="wp-image-3268 jetpack-lazy-image" data-lazy-src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-12.gif?resize=941%2C549&amp;is-pending-load=1#038;ssl=1" data-recalc-dims="1" height="549" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-12.gif?resize=941%2C549&amp;ssl=1" srcset="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="941"/>
   <noscript>
    <img alt="" class="wp-image-3268" data-recalc-dims="1" height="549" src="https://i0.wp.com/www.iyouport.org/wp-content/uploads/2019/05/0-12.gif?resize=941%2C549&amp;ssl=1" width="941"/>
   </noscript>
  </figure>
  <p class="graf graf--p">
   这里并没有中央平台；它只是一条消息，您可以将这些消息传播给其他人。作一个类比，就像我写了一张明信片并允许任何人进入我的房子并复制它，那些复制它的人允许其他人进入他们的房子进行复制（这实际上从未发生过，但你明白了）。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    版权法案的第13条不适用于分散化的 Web 基础结构，因为没有平台可以开始监控内容。
   </strong>
  </p>
  <p class="graf graf--p">
   这意味着第13条的范围有限，当然并不是说未来也依旧如此，对我而言，这表明我们需要更好、更可持续的版权改革建议。⚪️
  </p>
  <p class="graf graf--p graf--hasDropCapModel graf--hasDropCap">

    nternet Filtering is Obsolete in a Decentralized Web
   </a>
   . There is no central platform; it’s just a message you make available to others to copy onto their computer directly, which you allow them to spread to others.
  </p>
