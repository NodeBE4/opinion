  <hr/>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    嗨，我是 caasi ，很高興能幫這次的 civicth0n 整理各種資料。在這裡想談談我參與過的 g0v 黑客松，與其他黑客松的異同。
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    每次 g0v 黑客松舉辦兩週前，會在活動報名網站（類似活動行， g0v 用的是
    <a href="http://g0v-jothon.kktix.cc/events/g0v-hackath20n">
     <span class="s2">
      KKTIX
     </span>
    </a>
    ）開好活動，並預告何時可以報名（通常是週一中午）。報名當天，通常半小時內就會額滿。隨著活動的日子逼近，陸陸續續會有人取消報名，想參加但是沒有報到名的人可以等，或者主動在 Slack 上詢問有沒有人願意轉讓。
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    每次活動相關資訊會整理在獨立的共筆上，以便查閱。參加者於早上九點到活動場地報到，在聽完如何參與黑客松的介紹，與各個專案的三分鐘提案報告後，必須簡單介紹自己。 g0v 黑客松目的是解決問題，而 g0v是個強調「去中心化」的組織，故不主動幫參加者分組，也不藉著自我介紹時間讓參加者交流，於是自我介紹時不說「大家好」，只報自己的名字與專長。例如我會說：「我是卡西，專長是 JavaScript 、考古」（最多講三個專長），然後下一個人接著介紹。
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    這樣一來，提案者知道要找誰幫忙。隨著用餐和點心時間參與者之間彼此交流，自然地分組。活動場地是開放的，也不強求一天內一定要完成完整作品，於是並不存在明顯的分組邊界。也許會有人在這組幫忙整理了些資料，就去找其他組的朋友聊天；也許有人同時幫好幾組設計 UI 或是吉祥物；也許有人知道許多業界祕辛，整天都在講故事。
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    下午兩點半，會有一段自由報名的短講（類似 lightning talk ），這些短講不見得要和當天的提案有關，心得分享、技術分享都可以講。接著到了下午五點，專案各自花五分鐘報告今天的成果，最後參與者幫忙整理環境後，才離開活動場地。會後熟識的朋友也許會一起吃飯，同一個專案的組員也許會約好繼續完成專案的日子（稱為小松，好與稱為大松的 g0v 黑客松區別）。
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    對 g0v 的歷史有興趣，可以讀《零時政府的第一年》，然後到 http://hack.g0v.tw/ 看看過往活動紀錄。
   </span>
  </p>
  <p class="p1">

  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    下面列出一些讓我印象深刻的 g0v 專案：
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    長期專案
   </span>
  </p>
  <p class="p1">
   <span style="font-family: helvetica, arial, sans-serif;">
    <em>
     <strong>
      <span class="s1">
       福利請聽
      </span>
     </strong>
    </em>
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    福利請聽從 2013 年初第二次 g0v 黑客松（ hackath1n ，從 0 開始算）開始，到了 2013 年六月（ hackath3n ）才成形，一開始是聽障福利小幫手，後來成為全方位的社會福利搜尋網站。按 http://listening.g0v.tw 的說明，買房、生子、創業、生病都能搜尋。資料不只用爬蟲爬，也依賴手動編輯或由能提供福利的機構主動提供法條。
   </span>
  </p>
  <p class="p1">
   <span style="font-family: helvetica, arial, sans-serif;">
    <em>
     <strong>
      <span class="s1">
       NewsDiff &amp; 新聞小幫手
      </span>
     </strong>
    </em>
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    新聞網站常常修改發表出來的文章， NewsDiff 主動幫修改前跟修改後的文章備份，留下見證。 NewsDiff 是從 2013 中（ hackath3n ）開始的專案。在蒐集了一年多的資料後，甚至還開始分析新聞資料。兩個月後的黑客松（ hackath4n ）同一位開發者 ronnywang 開始了另外一個案子：新聞小幫手，來解決客觀新聞錯誤的問題。安裝新聞小幫手後，瀏覽到爭議新聞，畫面上將出現通知，甚至事後提醒之前看過的新聞有錯誤。半年後（ hackath7n ）繼續改進至今。
   </span>
  </p>
  <p class="p1">
   <span style="font-family: helvetica, arial, sans-serif;">
    <em>
     <strong>
      <span class="s1">
       萌典
      </span>
     </strong>
    </em>
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    萌典始於葉平教授開始的《還文於民》計畫，希望能將教育部重編國語辭典網站改造得更便於查詢。大概和第二次黑客松（ hackath1n ）同一時期。在一兩天內就將上千條詞條爬出來，交給網民們糾錯，整理成三顆星開放資料，之後演進成五星開放資料（可以參考 www 創始人 Tim Berners-Lee 提出的開放資料五顆星分類架構）。台灣曾經爭辯過政府開放資料到底該整理得多完整，萌典展示了三星資料就能成為網站、 iOS 、桌面字典等應用的基礎。後來又加入閩南語、英語、德語、法語、客家語、原住民阿美語，是唐鳳老師維護的有名專案之一。
   </span>
  </p>
  <p class="p1">
   <span style="font-family: helvetica, arial, sans-serif;">
    <em>
     <strong>
      <span class="s1">
       vTaiwan
      </span>
     </strong>
    </em>
   </span>
  </p>

  <p class="p1">
   <span style="font-family: helvetica, arial, sans-serif;">
    <em>
     <strong>
      <span class="s1">
       與展覽結合
      </span>
      <span class="s1">
       與社會交往的藝術的專案
      </span>
     </strong>
    </em>
   </span>
  </p>
  <p class="p1">
   <span class="s1" style="font-family: helvetica, arial, sans-serif;">
    g0v 曾獲邀參與“
    <strong>
     <em>
      與社會交往的藝術展”
     </em>
    </strong>
    ，展出資料可視化成果，當時的討論可以參考
    <a href="https://g0v.hackpad.com/-x-CGK7T9IuQIw">
     此共筆
    </a>
    。有趣的是，共筆中討論的是如何「真的」視覺化各個專案的成果，例如以硬幣表示長條圖、用氣球表現預算多寡、真的製造些 pm 2.5 。
