  <p class="graf graf--p">
   默认情况下，Google 会收集有关用户行为的大量数据，包括终身网络搜索历史记录、位置和 YouTube 观看记录。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%e5%88%b0%e5%ba%95%e6%9c%89%e5%a4%9a%e5%b0%91%e8%b7%9f%e8%b8%aa%e5%99%a8%e5%9c%a8%e7%9b%97%e7%aa%83%e4%bd%a0%e7%9a%84%e9%9a%90%e7%a7%81%ef%bc%9f%e8%b0%b7%e6%ad%8c%e4%bb%a5%e5%85%b6%e6%83%8a%e4%ba%ba/" href="https://www.iyouport.org/%e5%88%b0%e5%ba%95%e6%9c%89%e5%a4%9a%e5%b0%91%e8%b7%9f%e8%b8%aa%e5%99%a8%e5%9c%a8%e7%9b%97%e7%aa%83%e4%bd%a0%e7%9a%84%e9%9a%90%e7%a7%81%ef%bc%9f%e8%b0%b7%e6%ad%8c%e4%bb%a5%e5%85%b6%e6%83%8a%e4%ba%ba/" rel="noopener noreferrer" target="_blank">
     到底有多少跟踪器在盗窃你的隐私？谷歌以其惊人的跟踪量位居榜首
    </a>
    》
   </li>
  </ul>
  <p class="graf graf--p">
   在隐私方面的强烈反对和持续的监管威胁中，该公司已开始大肆宣传其最近发布的隐私工具，例如 能够自动删除他们所收集的有关您的某些数据，这些数据有助于推动其1,160亿美元的广告业务发展。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    别被他们骗了《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%e4%bb%a5%e9%9a%90%e7%a7%81%e4%b8%ba%e7%9b%ae%e7%9a%84%e7%9a%84%e6%96%97%e4%ba%89%e5%a6%82%e4%bd%95%e8%ae%a9%e4%bd%a0%e5%8f%98%e6%88%90%e4%ba%86%e7%9b%91%e8%a7%86%e7%9a%84%e7%9b%9f%e5%8f%8b/" href="https://www.iyouport.org/%e4%bb%a5%e9%9a%90%e7%a7%81%e4%b8%ba%e7%9b%ae%e7%9a%84%e7%9a%84%e6%96%97%e4%ba%89%e5%a6%82%e4%bd%95%e8%ae%a9%e4%bd%a0%e5%8f%98%e6%88%90%e4%ba%86%e7%9b%91%e8%a7%86%e7%9a%84%e7%9b%9f%e5%8f%8b/" rel="noopener noreferrer" target="_blank">
     以隐私为目的的斗争如何让你变成了监视的盟友
    </a>
    》
   </li>
  </ul>
  <p class="graf graf--p">
   通过访问
   <a class="markup--anchor markup--p-anchor" data-href="https://myaccount.google.com/activitycontrols" href="https://myaccount.google.com/activitycontrols" rel="noopener noreferrer" target="_blank">
    Google的活动控制页面
   </a>
   ，您可以选择清除连续三个月或18个月的这些历史数据。
  </p>
  <p class="graf graf--p">
   该公司推出了此工具，并具有通过Google活动页面手动删除数据的功能，这通常被认为是用户控制其隐私的多种方式之一。
  </p>
  <p class="graf graf--p">
   去年，两名 Google 产品经理在该公司的
   <a class="markup--anchor markup--p-anchor" data-href="https://www.blog.google/technology/safety-security/automatically-delete-data/" href="https://www.blog.google/technology/safety-security/automatically-delete-data/" rel="noopener noreferrer" target="_blank">
    博客中
   </a>
   写道：“您应该始终能够以最适合自己的方式管理您的数据；我们致力于为您提供最好的控制，以实现这一目标。”
  </p>
  <p class="graf graf--p">
   实际上，
   <strong class="markup--strong markup--p-strong">
    即使这些自动删除工具为 Google 带来了良好的公关效果，也无法为用户带来任何帮助
   </strong>
   。
  </p>
  <p class="graf graf--p">
   专家说，到三个月时，Google已经从用户数据中提取了几乎所有潜在价值，并且从广告的角度来看，当数据使用了几个月以上时，它们几乎变得一文不值了。
  </p>
  <p class="graf graf--p">
   数字广告公司 WPromote 付费搜索主管 David Dweck 说：“长达一个月的时间都是非常宝贵的。任何数据放置长达一个月就基本变得没价值了”。
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    越新越值钱
   </strong>
  </p>
  <p class="graf graf--p">
   Dweck 表示，在数字广告行业中，近期活动至关重要。
  </p>
  <p class="graf graf--p">
   例如，如果您开始在 Google 上搜索房地产信息或查找房屋价格，则Google就会将您放到广告客户的“潜在购房者”类别中。
  </p>
  <p class="graf graf--p">
   该信息对于房地产经纪人、评估师、和贷款方而言，是非常有价值的，广告定位会立即跟进，并且紧接着诸如油漆工或家电品牌之类的其他公司也会尝试跟进您的房屋购买，这些信息可能会保留一段时间。
  </p>
  <p class="graf graf--p">
   Dweck 表示，如果广告客户可以根据目标人几个月前的活动来定位用户，这是不寻常的。
  </p>
  <p class="graf graf--p">
   是有例外，例如，他引用电视网络针对棒球迷的例子，如果球迷一直在推特活跃，季后赛广告就会瞄准他们。但这种情况很少见。
  </p>
  <p class="graf graf--p">
   Dweck 说：“我觉得他们每三个月自动清理一次数据确实只是一项口头服务。” ；“这不是任何大的改变，因为事实是，没有人真正会购买过时数据。”
  </p>
  <p class="graf graf--p">
   此外，Dweck 说，
   <strong class="markup--strong markup--p-strong">
    从 Google 删除数据并不能阻止广告客户自己跟踪您
   </strong>
   。
  </p>
  <p class="graf graf--p">
   如果您在Google上搜索鞋子，然后点击进入某个鞋品牌的网站，那么无论谷歌是否清除自己的数据，该品牌都会跟踪你。
  </p>
  <p class="graf graf--p">
   该跟踪可能会与您使用的任何设备相关联，因此它会一直持续到您购买新手机或计算机。
  </p>
  <p class="graf graf--p">
   Dweck说：“一旦您在广告客户的网站上被盯上了，对我们来说90天的有效期就无关紧要了。”
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    监视资本家是不会改变的
   </strong>
  </p>
  <p class="graf graf--p">
   谷歌发言人只会说，它的研究表明用户偏爱三个月和十八个月的窗口期 ……就像政府说 “这是人民的意愿”。你知道这叫被代表。
  </p>
  <p class="graf graf--p">
   还要注意的是，尽管仍然需要用户访问三个单独的网页来分别进行删除，但Google仍然允许用户手动删除其所有活动数据。
  </p>
  <p class="graf graf--p">
   反跟踪扩展程序 Ghostery 的总裁兼产品和营销主管 Jeremy Tillman 不接受这样的说法，即 用户只希望按季度删除数据。
  </p>
  <p class="graf graf--p">
   他认为 Google 应该提供更短的自动删除期限，或者提供不保留任何活动数据的选项。
  </p>
  <p class="graf graf--p">
   他说：“就他们所做的一切，我认为，
   <strong class="markup--strong markup--p-strong">
    对于公关而言，这不过是为了让他们在受到审查时提供可靠的辩护
   </strong>
   。”
  </p>
  <p class="graf graf--p graf--startsWithDoubleQuote">
   “
   <strong class="markup--strong markup--p-strong">
    他们不可能做任何有意义的事，因为那会威胁到其潜在的商业模式
   </strong>
   。”
  </p>
  <p class="graf graf--p">
   Dweck 表示，即使只有24小时的活动数据，对广告客户来说也非常有价值。他仍然承认，每日的自动删除会严重影响广告客户基于搜索活动的定位能力。
  </p>
  <p class="graf graf--p">
   他说：“如果在开始的24小时内没有抓住您，我们将不会知道您的意图是什么。如果人们在24小时内擦除数据会对我们产生很大影响，因为我们将无法在短时间内将您作为目标。”
  </p>
  <p class="graf graf--p">
   <strong class="markup--strong markup--p-strong">
    三个月的先例
   </strong>
  </p>
  <p class="graf graf--p">
   Google 建立的自动删除窗口期可能为科技行业树立了先例。
  </p>
  <p class="graf graf--p">
   去年亚马逊也宣布了自己的自动删除工具，用于删除录制的Alexa语音命令，使用的工具与Google首次亮相的三个月和十八个月相同。（类似于 Google，Amazon 还允许用户通过其网站或Alexa应用程序手动删除其完整历史记录）
  </p>
  <p class="graf graf--p">
   亚马逊发言人说，该公司选择了这些时间范围，以便用户可以查看自己的活动，认为父母可能希望查询孩子的Alexa命令，或客户可能希望在进行故障排除时查阅他们的语音历史记录。
  </p>
  <p class="graf graf--p">
   与Google不同，Amazon不会直接使用Alexa活动进行广告定位。
  </p>
  <p class="graf graf--p">
   但是，如果您使用 Alexa 在 Amazon 上购物或与其他 Amazon 服务进行交互，则该活动反过来会影响该公司在其他地方显示给你的“建议” —— 简单说
   <strong class="markup--strong markup--p-strong">
    它一直在追踪你
   </strong>
   。
  </p>
  <p class="graf graf--p">
   即使您删除了相关的 Alexa 录音，Amazon 仍保留了这些数据，并且无法自动删除。
  </p>
  <p class="graf graf--p">
   更长的保留期不仅是针对性广告的动力。他们还为科技巨头提供了更多的时间来根据用户数据训练他们的机器学习模型。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%f0%9f%91%80%e4%ba%9a%e9%a9%ac%e9%80%8a%e5%92%8c%e8%b0%b7%e6%ad%8c%e6%ad%a3%e5%9c%a8%e5%b0%86%e6%96%b0%e7%9a%84%e7%9b%91%e8%a7%86%e5%92%8c%e7%aa%83%e5%90%ac%e8%ae%be%e5%a4%87%e5%a1%9e%e8%bf%9b/" href="https://www.iyouport.org/%f0%9f%91%80%e4%ba%9a%e9%a9%ac%e9%80%8a%e5%92%8c%e8%b0%b7%e6%ad%8c%e6%ad%a3%e5%9c%a8%e5%b0%86%e6%96%b0%e7%9a%84%e7%9b%91%e8%a7%86%e5%92%8c%e7%aa%83%e5%90%ac%e8%ae%be%e5%a4%87%e5%a1%9e%e8%bf%9b/" rel="noopener noreferrer" target="_blank">
     亚马逊和谷歌正在将新的监视和窃听设备塞进您的居室和汽车里
    </a>
    》
   </li>
  </ul>
  <p class="graf graf--p">
   由于亚马逊、谷歌和其他公司使用人类来审查语音助手录音的准确性而受到强烈反对，这本身已成为严重的隐私问题。
  </p>
  <ul class="postList">
   <li class="graf graf--li">
    《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%e4%ba%9a%e9%a9%ac%e9%80%8a%e5%90%91%e9%94%99%e8%af%af%e7%9a%84%e4%ba%ba%e5%8f%91%e9%80%81%e4%ba%86%e7%ba%a61700-%e6%9d%a1-alexa-%e5%bd%95%e9%9f%b3/" href="https://www.iyouport.org/%e4%ba%9a%e9%a9%ac%e9%80%8a%e5%90%91%e9%94%99%e8%af%af%e7%9a%84%e4%ba%ba%e5%8f%91%e9%80%81%e4%ba%86%e7%ba%a61700-%e6%9d%a1-alexa-%e5%bd%95%e9%9f%b3/" rel="noopener noreferrer" target="_blank">
     亚马逊向错误的人发送了约1,700 条 Alexa 录音
    </a>
    》
   </li>
   <li class="graf graf--li">
    《
    <a class="markup--anchor markup--li-anchor" data-href="https://www.iyouport.org/%e4%b8%80%e8%b5%b7%e5%ae%98%e5%8f%b8%e5%a6%82%e4%bd%95%e6%8f%ad%e7%a4%ba%e4%ba%86%e4%ba%9a%e9%a9%ac%e9%80%8a-alexa-%e7%9a%84%e7%9c%9f%e9%9d%a2%e7%9b%ae%ef%bc%9f/" href="https://www.iyouport.org/%e4%b8%80%e8%b5%b7%e5%ae%98%e5%8f%b8%e5%a6%82%e4%bd%95%e6%8f%ad%e7%a4%ba%e4%ba%86%e4%ba%9a%e9%a9%ac%e9%80%8a-alexa-%e7%9a%84%e7%9c%9f%e9%9d%a2%e7%9b%ae%ef%bc%9f/" rel="noopener noreferrer" target="_blank">
     一起官司如何揭示了亚马逊 Alexa 的真面目？
    </a>
    》
   </li>
  </ul>
  <p class="graf graf--p">
   亚马逊承认，通过保持 Alexa 活动更长的时间，它就有更多的时间来训练其机器学习模型。
  </p>
  <p class="graf graf--p">
   该公司表示：“客户期望 Alexa 会随着时间的推移而改善，并接受来自不同客户的真实需求对 Alexa 进行培训，有助于 Alexa 对客户的语音模式、方言、口音和词汇的变化做出正确的响应。”
  </p>
  <p class="graf graf--p">
   但这并不能解释
   <strong class="markup--strong markup--p-strong">
    为什么用户无法选择是否愿意根据需要自动删除其活动
   </strong>
   。
  </p>
  <p class="graf graf--p">
   亚马逊和谷歌都提供了控制面板，也许足以抵制类似于欧洲GDPR法规的严格规定，但他们不愿放弃任何可能影响其监视资本主义业务方式的东西。
  </p>
  <ul class="postList">
